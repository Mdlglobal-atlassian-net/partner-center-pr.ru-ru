# <a name="new-commerce-experience-in-csp---azure-billing"></a>Новый интерфейс для коммерческих приложений в CSP. Выставление счетов в Azure 

Выставление счетов в рамках плана Azure — это простой инструмент выставления счетов за счет использования согласованной даты выставления счетов и расчетного периода на основе календарного месяца. Сведения о платформе выставления счетов см. в статье [Partner Center Modern Commerce Operating Guide](https://assetsprod.microsoft.com/mpn/Partner-Center-Modern-Commerce-Operating-Guide.docx) (Руководство по современной коммерции с помощью Центра партнеров).

## <a name="summary-of-billing-essentials"></a>Основные сведения о выставлении счетов

- **Дата выставления счета-фактуры**. Счет-фактура и файл выверки будут доступны с помощью API или Панели мониторинга Центра партнеров в полночь по UTC.

- **Период выставления счетов**. Период выставления счетов определяется как за календарный месяц, например 1.10–31.10 или 1.11–30.11.

- **Периоды начисления оплаты**: Плата будет начисляться за календарный месяц. Например, если партнер, выставляющий счета, добавляет службы Azure через план Azure 15.10, а клиент начинает использование служб Azure с 15.10, то партнер получит счет-фактуру или файл выверки 8.11 по использованию клиентом служб в течение периода обслуживания 15.10–31.10. Счет за следующий месяц, который будет создан на 8.12, содержит все расходы за период обслуживания 1.11–31.11.

- **Срок оплаты счета-фактуры**. В течение 60 дней.

- **Валюта счета**. Партнеры будут по-прежнему получать счета-фактуры в назначенной валюте страны клиента. Например, если партнер в Ирландии выставляет счета клиентам в Великобритании, Норвегии и Германии, то партнер будет получать счет-фактуру или файл выверки в фунтах стерлингов (GBP), норвежских кронах (NOK) и евро (EUR).

- **Поощрения для партнеров**. Оплачено 45 дней с конца месяца счета-фактуры.

##  <a name="access-your-invoices-and-recon-files"></a>Доступ к своим счетам-фактурам и файлам выверки

Глобальный администратор или администратор выставления счетов для вашей компании получат электронное сообщение, когда счет-фактура будет готов к просмотру. 

**Доступ к файлу выверки и счету-фактуре**

1. Войдите в Центр партнеров.

2. В меню Центра партнеров выберите **Billing** (Выставление счетов).

3. Выберите вкладку для параметра **Calendar-based** (Календарный) и валют, которые вас интересуют.

![выставление счетов](images/azure/billing1.png)

4. Выберите **Invoice and Recon file** (Счет-фактура и файл выверки).  

Чтобы просмотреть предыдущие счета-фактуры и файлы выверки, разверните строку "История счетов" ниже.

## <a name="read-the-invoice"></a>Чтение счета-фактуры

1. Счет будет доступен не позднее 8 числа каждого месяца.

2. У партнеров есть 60 дней для предъявления платежа.

3. Расчетный период охватывает заданный календарный месяц, например 1.10–31.10.

4. Плата взимается в процентах от корректировок (сумма — это доля от "Партнерского кредита на управляемые службы").

5. Чтобы получить дополнительные сведения о выставлении счетов, см. файл выверки счета-фактуры и файл ежедневного использования.

![счет-фактура](images/azure/invoice1.png)

## <a name="read-the-recon-file"></a>Чтение файла выверки

1. Каждый счетчик подписки Azure может иметь до двух строк выставления счетов в файле выверки.

2. Если счетчик для любого типа скидки или кредита (например, для скидок уровня 1 или партнерского кредита для управляемых служб) действует на протяжении всего календарного месяца, то файл выверки будет содержать только одну строку выставления счета. Столбец **PriceAdjusmentDescription**  будет содержать скидку или полученный кредит; эффективная цена за единицу будет считаться розничной ценой за вычетом партнерского кредита или любых других скидок.

3. Если в течение всего календарного месяца счетчик не был назначен для партнерского кредита для управляемых служб, то файл выверки будет содержать только одну строку выставления счета, а действительная цена за единицу будет розничной ценой (которая будет ценой за единицу).

4. Если счетчик назначен для **Партнерского кредита для управляемых служб** за часть месяца, файл выверки будет содержать две строки выставления счета. Одна строка будет представлять дни, в которых счетчик действует, а вторая — дни, в которые счетчик не применяется. 

## <a name="read-the-daily-usage-file"></a>Чтение файла ежедневного использования

- Счетчики подписки в плане Azure оцениваются и распределяются ежедневно. 

- **Партнерский кредит для управляемых служб** определяется и применяется ежедневно.

- Каждый счетчик подписки будет иметь строку для каждого дня месяца, когда служба использовалась.

- См. пример ниже.

  - Счетчик действует для **Партнерского кредита для управляемых служб** за период 1.07–3.07 (обратите внимание, что цена за единицу является розничной ценой за вычетом партнерского кредита).

   - Счетчик не действует для **Партнерского кредита для управляемых служб** за период 4.07–7.07 (обратите внимание, что цена за единицу является розничной ценой).

    - Счетчик действует для **Партнерского кредита для управляемых служб** за период 8.07–31.07 (обратите внимание, что цена за единицу является розничной ценой за вычетом партнерского кредита).

## <a name="invoice-in-customer-currency"></a>Счет-фактура в валюте клиента 

Плата за службы Azure в плане Azure будет взиматься в долларах США (USD) и оплачиваться в валюте назначенной страны клиента. Если счет выставляется не в долларах США (USD), то на последней странице счета будет указан используемый курс валют. Курс валют определяется за месяц и применяется к следующему счету. Полный список валют см. в [New commerce offers country availability and customer currency matrix](https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE3Qn1V) (Таблица валют клиентов и доступности новых коммерческих предложений в разных странах). 

Корпорация Майкрософт будет использовать данные [Thompson Reuters](https://developers.thomsonreuters.com/content/wm-company) при определении курса валют, используемого для определения и конвертации валюты счета. Курс валют будет обновляться и доступен за день до первого дня месяца, для которого он применяется.

**Пример**:  Плата за использование в период обслуживания с 1 августа до 31 августа будет взиматься по курсу валют, опубликованному 1 августа. Эта плата отобразится в сентябрьском счете-фактуре, и курс валют будет указан на последней странице счета-фактуры. 

Пользователи-арендаторы партнера будут по-прежнему видеть связанные с ролью сведения обо всех клиентах и заказах, независимо от валюты выставления счета. Кроме того, такой пользователь сможет видеть все счета во всех валютах.  
 
## <a name="azure-reservations"></a>Резервирования Azure 

В случае приобретения [резервирования Azure](https://docs.microsoft.com/partner-center/azure-reservations) с помощью плана Azure, сначала можно будет выбрать разовое выставление счета в Центре партнеров. Ежемесячное выставление счетов доступно на портале Azure. Ежемесячное выставление счетов будет доступно в Центре партнеров позже. 

## <a name="azure-cost-management"></a>Azure Cost Management 

Средства Azure Cost Management помогут организациям визуализировать и оптимизировать свои затраты в Microsoft Azure и управлять ими. Этот компонент будет доступен на портале Azure. У партнеров будет постоянно доступное решение с низкой задержкой, в котором предусмотрены следующие функции. 

- Расширенные средства анализа и оповещения о бюджете 
- Интерфейсы API и соединители Power BI 
- Представление с несколькими клиентами 
- Бесплатное управление затратами на Azure 
- Расширение ролей и пользователей 

Дополнительные сведения об этой функции, которая стала доступна для корпоративных соглашений в феврале 2019 г., см. в разделе [Azure Cost Management](https://azure.microsoft.com/services/cost-management). Эта возможность доступна только в службах Azure, приобретенных в рамках этого нового коммерческого предложения Azure в CSP. 
 
## <a name="azure-spending"></a>Расходы на Azure 

Средство управления затратами Azure будет доступно в Центре партнеров для новых коммерческих предложений в CSP. После применения эта возможность позволит партнерам видеть:  

- итоговый бюджет клиента; 
- общую оценку расходов на существующий план Azure; 
- процент использования служб клиентами за каждый расчетный период. 

Так как модель выставления счетов для служб Azure в плане Azure — это использование служб после оплаты, чтобы избежать выставления счета, размер которого превышает ожидаемый, партнеры могут формировать месячный бюджет и следить за ростом использования служб. Бюджет может быть применен к одному клиенту или нескольким клиентам одновременно.
