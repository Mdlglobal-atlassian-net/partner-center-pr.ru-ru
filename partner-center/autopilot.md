# <a name="customize-a-devices-out-of-box-experience-with-windows-autopilot-profiles"></a>Настроить out-of-box устройства с помощью профилей Windows Autopilot

**Область применения**

- Партнерам CSP счет direct, непрямые поставщики и непрямые торговые посредники

Если вы управляете устройства пользователей, может потребоваться настроить out-of-box (OOBE) для клиента. Можно предварительной конфигурации новых устройств с помощью профилей Windows Autopilot перед доставкой устройств для пользователей и применить новые профили для устройств, которые уже приобрели клиентов. В этой статье объясняется, как создавать и применять профили Autopilot на устройствах в центре партнеров.

Если вы еще не знакомы с помощью Autopilot, просмотрите сведения в следующих статьях:

- [Обзор Windows Autopilot](https://docs.microsoft.com/windows/deployment/windows-10-auto-pilot)
- [Справочник развертывания AutoPilot](http://assetsprod.microsoft.com/autopilot-deployment-program-reference-guide-csp.docx)  

## <a name="overview"></a>Обзор

С помощью компонента Windows Autopilot в центре партнеров вы можете создать пользовательские профили, чтобы применить к устройствам клиента. Следующие параметры профиля были доступны на момент публикации этой статьи:

- Пропуск параметров конфиденциальности. Это необязательный параметр профиля Autopilot позволяет организациям не Узнайте о параметрах конфиденциальности во время запуска при первом Включении.

- Отключение создания учетной записи локального администратора на устройстве. Организации могут выбирать, будут ли настройки устройства пользователь должен иметь доступ с правами администратора, после завершения процесса.

- Автоматически настроить устройство для работы или учебного заведения. Все устройства, зарегистрированные с помощью Autopilot автоматически будет считаться рабочих или учебных устройств, поэтому этот вопрос не придется во время запуска при первом Включении.

- Пропустить страницы регистрации настройки Кортаны, OneDrive и OEM. Все устройства, зарегистрированные с помощью Autopilot автоматически пропустит одной страницы на другую в процессе out-of-box experience (OOBE).

- Пропустите лицензионное соглашение (EULA). Начиная с Windows 10 версии 1709, организации могут решили пропустить страницу лицензионного соглашения, отображаются во время запуска при первом Включении. См. в разделе [Windows Autopilot отказе](#windows-autopilot-eula-dismissal) ниже важные сведения необходимо учитывать о пропуск странице Лицензионное соглашение во время установки Windows.

Примените следующие разрешения профиля и устройства управления и ограничения:

- Партнеры CSP могут продолжать управлять профилями Autopilot существующих клиентов, с которыми они отношения торгового посредника, даже если клиенты отобрали делегированные права администратора партнера.

- Можно управлять имеющимися устройствами за ваших клиентов, которые были добавлены вами или другим партнером CSP.

- Вы не можете управлять устройствами, которые загрузил ваш клиент магазин Майкрософт для бизнеса или портала Microsoft Intune.

## <a name="create-and-manage-autopilot-profiles-in-partner-center"></a>Создание и управление профилями Autopilot в центре партнеров

В центре партнеров можно создать развертывания Windows AutoPilot и применить их к устройствам.

>[!NOTE]
>Только агенты можно создавать и применять профили.

### <a name="create-a-new-autopilot-profile"></a>Создайте новый профиль Autopilot

1. Выберите **клиентов** в меню центра партнеров, а затем выберите клиента, которого вы создаете профиль Autopilot.

2. На странице сведений о клиенте выберите **устройства**.

3. **Профили Windows Autopilot** установите **Добавить новый профиль**.

4. Введите имя и описание профиля, а затем настройте параметры OOBE. Выберите:  

   - Пропуск параметров конфиденциальности при настройке

   - Отключение учетной записи локального администратора при настройке
  
   - Автоматический пропуск страниц при настройке<br>
        (Включает *Автоматический выбор конфигурации для работы или учебного заведения* и *Пропустить Кортаны, OneDrive и OEM страницы регистрации для установки*)
  
   - Пропустите лицензионное соглашение (EULA)<br> 
       >[!IMPORTANT] 
       >См. в разделе [Windows Autopilot отказе](#windows-autopilot-eula-dismissal) ниже важные сведения необходимо учитывать о пропуск странице Лицензионное соглашение во время установки Windows.

5. По завершении нажмите кнопку **Отправить**.

### <a name="apply-an-autopilot-profile-to-customer-devices"></a>Примените профиль Autopilot на устройствах пользователей

>[!NOTE]
>Приведенные ниже инструкции предполагается, что вы уже добавили устройств клиента в центр партнеров и что есть доступ к их списка устройств. Если вы еще не добавили устройств клиента, следуйте инструкциям в [устройствах добавить учетную запись клиента](#add-devices-to-a-customers-account) , а затем выполните указанные ниже действия.

После создания профиля Autopilot для клиента, его можно применить к устройствам клиента.

1. Выберите **клиентов** в меню центра партнеров, а затем выберите клиента, которого вы создали профиль Autopilot.

2. На странице сведений о клиенте выберите **устройства**.

3. В группе **Профили применить к устройствам** выберите устройства или группы устройств, которые вы хотите добавить профили для, а затем выберите **применить профиль**. Профиль, который вы только что применили отображается в столбце " **профиль** ".

4. Выполните следующие действия, чтобы убедиться, что профиль будет успешно применена к устройству.

    А.  Подключите устройство к сети и включите его.

    b.  Убедитесь, что отображаются соответствующие экраны OOBE (если они должны отображаться).

    c.  При остановке процесс запуска при первом Включении, сбросьте устройство к заводским настройкам по умолчанию для подготовки для нового пользователя.

### <a name="remove-an-autopilot-profile-from-a-customers-device"></a>Удалить профиль Autopilot на устройстве пользователя

1. Выберите **клиентов** в меню центра партнеров, а затем выберите клиента, которого вы создали профиль Autopilot.

2. На странице сведений о клиенте выберите **устройства**.

3. В группе **Профили применить к устройствам** выберите устройства, которые вы хотите удалить профиль, а затем выберите **Удалить профиль**.

   >[!NOTE]
   >Удаление профиля с устройства не приводит к удалению профиля из списка. Если вы хотите удалить профиль, следуйте инструкциям в [обновление или удаление профиля Autopilot](#update-or-delete-an-autopilot-profile).

### <a name="update-or-delete-an-autopilot-profile"></a>Обновление или удаление профиля Autopilot

Если пользователь может изменить работу out-of-box после того, как поставки устройств на них, вы можете изменить профиль в центре партнеров.

При подключении устройства пользователя к Интернету, оно будет последняя версия профиля во время запуска при первом Включении. Кроме того любое время, клиент восстановит устройство к заводским настройкам по умолчанию, устройство будет снова последняя версия профиля во время запуска при первом Включении.

1. Выберите **клиентов** в меню центра партнеров, а затем выберите клиента, желающего вам потребуется изменить профиль Autopilot.

2. На странице сведений о клиенте выберите **устройства**.

3. В разделе **профили Windows Autopilot** выберите профиль, который необходимо обновить. Внесите необходимые изменения, а затем выберите **Отправить**.

Для удаления этого профиля, выберите **Удалить профиль** в верхнем правом углу страницы.

### <a name="add-devices-to-a-customers-account"></a>Добавление устройств для учетной записи клиента

>[!NOTE]
>Агентов по продажам и агенты можно добавить устройства для учетной записи клиента.

Прежде чем применять настраиваемые профили Autopilot на устройствах пользователей, необходимо будет получать доступ к списку устройств клиента.

Если вы планируете использовать имя изготовителя Оборудования, серийный номер и комбинации модели, следует учитывать следующие ограничения:

- Эта кортежа работает только для новых устройств (4 k хэши, например) и не поддерживается для хэши 128b (RS2 и предыдущие устройства).

- Регистрация кортежа чувствителен к регистру, поэтому данные в файле должен модели и производителя имена ***точно*** предоставляется поставщиком Оборудования (поставщик оборудования).

Следуйте приведенным ниже инструкциям для добавления устройств для учетной записи клиента в центре партнеров.

1. Выберите **клиентов** в меню центра партнеров, а затем выберите клиента, которого вы хотите управлять устройствами.

2. На странице сведений о клиенте выберите **устройства**.

3. В группе **Профили применить к устройствам** выберите **Добавить устройства**.

4. Введите имя для списка устройств, а затем выберите **Обзор** , чтобы передать список клиента (в формат CSV-файла) в центр партнеров.

    >[!NOTE]
    >Вы должны были получить этот CSV-файл с помощью покупки устройства. Если вы не получаете CSV-файл, можно создать один самостоятельно, выполнив действия, описанные в [Добавление устройств Windows Autopilot](https://docs.microsoft.com/windows/deployment/windows-autopilot/add-devices#collecting-the-hardware-id-from-existing-devices-using-powershell).  

5. Отправка CSV-файл, а затем выберите **Сохранить**.

Если вы получите сообщение об ошибке при попытке загрузки CSV-файла, проверьте формат файла. Можно использовать только хэш оборудования или имя изготовителя Оборудования, серийный номер и модель (в указанном порядке столбов) или код продукта Windows. Также можно использовать образец CSV-файла, предоставляемый по ссылке рядом с **Добавить устройства** для создания списка устройств.

## <a name="windows-autopilot-eula-dismissal"></a>Windows Autopilot отказе

### <a name="important-information"></a>ВАЖНАЯ ИНФОРМАЦИЯ

Windows Autopilot позволяет настроить пользовательские установки Windows на устройствах, которыми вы управляете для своих клиентов. Если для этого клиента, можно отключить или скрыть некоторые экраны, которые обычно отображаются для пользователей при настройке Windows, включая экран принятия лицензионного соглашения (лицензионного соглашения).

Используя эту функцию, вы соглашаетесь, что отключение или скрытие экранов, предназначенных для предоставления пользователям уведомления или принятие условий, означает, что вы получили достаточное согласие и от своего клиента для скрытия условия и что вы лица Ваш клиент (ли организации или отдельного пользователя может быть), согласие на любые уведомления и принимаете любые условия, применимые к вашему клиенту. Это включает согласие с условиями лицензии или уведомления, которое отображалось бы для пользователя, если не бы вы не отключили или не скрыли его с помощью этого средства. Ваш клиент не может использовать программное обеспечение Windows на этих устройствах, если ваш клиент не приобрел законным способом лицензию на программное обеспечение у Майкрософт или лицензированных дистрибьюторов.
