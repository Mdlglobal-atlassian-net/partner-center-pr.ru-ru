## <a href="" id="processfaq"></a>Process

**Что такое новый процесс для подтверждения того, клиент принятия соглашение Microsoft Cloud (MCA)?**

Корпорация Майкрософт стремится помогает своим клиентам и партнерам сохраните соответствие нормативным требованиям. Как партнер в программе поставщиков облачных решений (CSP) Майкрософт вы несете ответственность за проверку Майкрософт положений и условий, включая допустимого использования продуктов и служб с клиентами Майкрософт. Чтобы лучше помочь партнерам в обеспечении соответствия нормативным требованиям и прозрачности мы включается новый процесс, требующий партнеры CSP, чтобы убедиться, что своим клиентам приняли условия MCA. Начиная с 7 августа 2018 г. мы внедрили новые поля в информационной панели центра партнеров пользовательском интерфейсе также API центра партнеров для партнерам для предоставления этих сведений. Изначально он будет необязательно для предоставления этого подтверждения, но он потребуется 7 ноября 2018 г. 

**Если это требование вступают в силу?**

Партнеры могут начинаться подтверждение принятия сразу же функциональность доступна 7 августа 2018 г. Начиная с 7 ноября 2018 г., информационной панели центра партнеров от пользователей требуется для подтверждения клиента принятие MCA на всех новых операций. 22 марта 2019 года API центра партнеров пользователям потребуется сделать то же самое. Крайний срок для пользователей API была расширена для них дополнительные раз для технические реализации.

**НОВЫЕ | Майкрософт проверить точность сведений клиента, предоставляемые партнеров?**

Майкрософт проверяет, что указанный адрес электронной почты клиента имеет правильный формат. Обязано партнера предоставление точной информации и информации, которая находится к неточным. 

**НОВЫЕ | Как можно партнера подтвердить согласие клиента из MCA, ли он подписан, задокументированы, это, или на бумаге?**

Партнер ответственность за соответствие, что пользователь принял условия MCA для управления своими процесса и записи. С помощью информационной панели центра партнеров или API центра партнеров процесса, партнер подтверждения того, что пользователь принял условия MCA.

**НОВЫЕ | Как корпорация Майкрософт проверит, подтверждения процесс будет завершен для клиента?**

Начиная 7 ноября 2018 г., когда партнер CSP запускает транзакцию, с помощью информационной панели центра партнеров для новых или существующих клиентов, систем Майкрософт автоматически проверит сведения о подтверждении клиент при наличии в центре партнеров Панель мониторинга. Если его нет, транзакции не смогут быть завершена. 

Начиная с 22 марта 2019 г., когда партнер CSP инициирует транзакции для новых или существующих клиентов с помощью API центра партнеров, систем Майкрософт автоматически проверит при наличии подтверждения сведений о клиенте. Если его нет, транзакции не смогут быть завершена. 

**НОВЫЕ | Уведомление, отправленное клиентам, чьи адреса электронной почты предоставляется в процессе подтверждения?**

Нет ни один процесс уведомления.

## <a href="" id="policyfaq"></a>Политика

**Почему Microsoft требующих партнеров, чтобы убедиться, что пользователи приняли MCA?** 

Соглашение Microsoft Cloud (MCA) защищает партнерами, клиентами и Microsoft, предоставляя права на использование клиента и условия, предоставленные для Microsoft Cloud заказов продуктов и служб. 

Правильного выполнения MCA защищает наших партнеров, клиентов и корпорации Майкрософт, обеспечивая взаимной выравнивание на многие важные разделы, в том числе, но не ограничиваясь безопасности, конфиденциальности и защиты данных. Процесс подтверждения гарантирует прозрачность и выравнивания и позволяет Майкрософт, чтобы лучше помочь клиентам, которым необходимо незамедлительно реагировать на запросы нормативным.

**Какие сведения о клиенте необходим для завершения процесса подтверждения?**
Следующие сведения является обязательным в отношении лица, приняли соглашение:
-   Дата принятия соглашения
-   Имени и фамилии
-   адрес электронной почты;
-   Номер телефона (необязательно) 

**НОВЫЕ | Что произойдет, если мы не соответствует срок?**

Партнеры могут начинаться подтверждение принятия сразу же функциональность доступна 7 августа 2018 г. Начиная с 7 ноября 2018 г., информационной панели центра партнеров от пользователей требуется для подтверждения клиента принятие MCA на всех новых операций. 22 марта 2019 года API центра партнеров пользователям потребуется сделать то же самое.

Во время применения новых покупок для существующего клиента или изменения в число количество на существующих подписок будет запрещено до принятия клиента подтвержден. Для нового клиента транзакции будет запрещено до принятия подтвержден. Это требование не будет снижена автоматическое продление.

**Области, которые это требование относится к?**

Подтверждение принятия клиента для MCA является обязательным для всех регионов в общедоступном облаке Microsoft. Подтверждение принятия клиента с помощью национальных облаках (включая Microsoft Cloud для правительства США и Microsoft Cloud в Германии) не поддерживается.

**Какие партнеры несут ответственность за подтверждение принятия клиента для MCA?**

Вступления в силу 7 ноября 2018 г. поставщиков облачных служб (CSP) для косвенных поставщиков и прямой CSP выставления счетов партнерам обе ответственность за подтверждение принятия клиента и могут с помощью API центра партнеров или информационной панели центра партнеров. Непрямые поставщики может потребоваться сначала собирать необходимую информацию от своих торговых посредников и затем сделать его доступным в корпорацию Майкрософт через API центра партнеров или информационной панели центра партнеров. От лица это косвенного поставщика, который имеет привязку контракта с корпорацией Майкрософт, чтобы убедиться, что для MCA приняли торговых посредников CSP конечным пользователям. 

**Какие сценарии клиента требуют подтверждения клиента принятие MCA?**

Принятие соглашения Microsoft Cloud необходимо подтвердить, когда партнеры CSP необходимо внести следующие операции:
1.  Нового клиента завершает работу новых покупок
2.  Существующего клиента завершает работу новых покупок
3.  Существующего клиента обновляет количество рабочих мест в существующие подписки

**Можно ли подтверждать принятия существующих клиентов, отдельно от своих новых покупок или число добавления?**

Да. Для всех пользователей, распространяется MCA расположение можно проверить принятия клиента.

**Каковы требования для клиентов, имеющих несколько партнеров CSP?**

Подтверждение принятия подтверждено партнера. Если клиент занимаются с несколькими партнерами CSP, каждого партнера должен будет подтвердить принятия.

**Я являюсь работа с непрямыми поставщиками CSP торгового посредника CSP. Что такое Мои ответственности?**

От лица это косвенным поставщиком, имеющий контракта с корпорацией Майкрософт, чтобы убедиться, что конечному клиенту принял MCA привязки. Однако косвенные поставщики могут иметь отдельный схем положений с непрямыми торговыми посредниками собирать принятия клиента. В этом случае косвенным поставщикам необходимо сначала собрать сведения принятия нужного клиента из своих торговых посредников и затем сделать его доступным в корпорацию Майкрософт через API центра партнеров или информационной панели центра партнеров. Этот процесс может поддерживать незамедлительно благодаря необходимую информацию для вашего косвенных поставщиков по их запросу.

**Я являюсь непрямым поставщиком CSP. Требуется для координации с моих торгового посредника CSP для подготовки?**

Рекомендуется связи этого требования для своих торговых посредников немедленно для координации доставку и получение информации о косвенных поставщиков, которые необходимо собрать сведения принятия нужного клиента из своих торговых посредников.

**Как можно ли объяснить моих клиентов, они должны проверить их принятия MCA?**

Правильного выполнения MCA защищает Microsoft, ее партнеров и клиентов за счет взаимной выравнивание на многие важные разделы, в том числе, но не ограничиваясь безопасности, конфиденциальности и защиты данных. Процесс подтверждения гарантирует прозрачность и выравнивания и позволяет Майкрософт, чтобы лучше помочь клиентам, которым необходимо незамедлительно реагировать на запросы нормативным.

**НОВЫЕ | Если клиент работает с одной непрямым поставщиком CSP, но транзакции выполняются с помощью нескольких CSP торговых посредников, каждого торгового посредника CSP требуется ли предоставить сведения о подтверждении непрямым поставщиком CSP?**

Косвенный поставщик CSP отвечает за предоставление подтверждения принятия MCA клиента в корпорацию Майкрософт. Для каждой учетной записи клиента только один подтверждения могут быть записаны в информационной панели центра партнеров за один раз. 

Для подтверждения того, что клиент контактное лицо ответственность за согласие MCA сделано, независимо от количества CSP торговых посредников, которым они взаимодействуете отвечает косвенного поставщика CSP.

**НОВЫЕ | Кто должен быть авторизованный пользователь для подтверждения принятия MCA внутри организации клиента?**

Обязано партнера для получения клиента принятие MCA от лица внутри организации клиента. Клиента обязано определить, кто авторизован для предоставления принятия. 

**НОВЫЕ | Какая версия MCA являются партнерами ответственность для подтверждения того, сообщающих о принятии клиентов? Партнеры нужно указать, какую версию MCA принят пользователем?**

Партнеры несут ответственность за использование самой MCA при создании транзакциям CSP с помощью нового клиента. Партнерам необходимо подтвердить принятия наиболее актуальная MCA при обновлении, изменение или завершение работы новых операций с существующим клиентом. 

В случае, что MCA обновляется до продления партнеры не должны иметь клиент немедленно повторно получить клиент принятие MCA, но должен повторно подтвердить согласие на MCA до продления. Тем не менее вы можете принять новые термины до обновления, чтобы воспользоваться преимуществами новых и улучшенных условия в MCA клиенту.

**НОВЫЕ | Я являюсь торгового посредника CSP. Если клиент принятие MCA записать в моей компании записей, нужно ли также предоставить эту информацию через информационную панель центра партнеров или API центра партнеров?**

От лица это косвенным поставщиком, имеющий контракта с корпорацией Майкрософт, чтобы убедиться, что конечному клиенту принял MCA привязки. Однако косвенные поставщики могут иметь отдельный схем положений с непрямыми торговыми посредниками собирать принятия клиента. В этом случае косвенным поставщикам необходимо сначала собрать сведения принятия нужного клиента из своих торговых посредников и затем сделать его доступным в корпорацию Майкрософт через API центра партнеров или информационной панели центра партнеров.

**НОВЫЕ | Что делать, если мой клиент не согласны с условиями MCA?**

Как партнер в программе CSP корпорации Майкрософт вы несете ответственность за принятие клиента MCA до размещения заказа. Это не новое требование. По любой причине с согласия клиента больше не с MCA, партнер должен отменить подписок, связанных с этого клиента.

**НОВЫЕ | Обеспечить торгового посредника согласие от имени клиента**

Нет.

**После убедитесь, что пользователь предоставил принятия, нужно ли повторно подтвердить его для клиента в будущем?**

После подтверждения клиент принял MCA, эта информация сохранится в разделе учетная запись клиента в центре партнеров, и нет необходимости повторно подтвердить принятия в течение всего времени, клиент рассматривается условия соглашения. Только должен повторно подтвердить принятия перед их размещением новый заказ.

В случае, что MCA обновляется до продления партнеры не должны иметь клиент немедленно повторно получить клиент принятие MCA, но должен повторно подтвердить согласие на MCA до продления. Тем не менее вы можете принять новые термины до обновления, чтобы воспользоваться преимуществами новых и улучшенных условия в MCA клиенту.

**НОВЫЕ | Я не себя комфортно, предоставляя корпорации Майкрософт Мои конечному клиенту пространств контактные данные. Как Майкрософт использует эти сведения?**

Когда партнер создает клиенте пользователя, она необходима для предоставления основное контактное лицо для клиента. Соглашение с Интернет-служб «Microsoft будет обработку личных данных только на приведенные инструкции от клиента.»

Партнеры, которые используют платформы, которые позволяют интегрировать API центра партнеров рекомендуется говорите непосредственно с поставщиками относительно свои планы и сроками для интеграции функций API центра партнеров, автоматизирует процесс подтверждения клиента.

Текущий API центра партнеров очень простой и не предоставляет для партнеров, чтобы программным образом получить копию MCA, а также записи версиях MCA, принимаемый клиента в рамках подтверждения. 

**НОВЫЕ | С помощью API центра партнеров для этого процесса требуется параметра userID. Как управлять этим требованиям, помещением заказов через платформ сторонних поставщиков, не интегрируется с Azure AD?**

В этом случае userID идентифицирует сотрудника в рамках клиента партнера, кто предоставление подтверждения согласия клиента. Если сотрудник не имеет соответствующий учетной записи пользователя Azure AD в рамках клиента партнера, то рекомендуется назначить учетной записи пользователя для этой цели и создания процесса по отдельности записать удостоверение сотрудника, предоставляя Подтверждение. 

**НОВЫЕ | Почему Microsoft требующих партнеров подтвердить согласие клиента на MCA вместо получение непосредственно из клиента?**
