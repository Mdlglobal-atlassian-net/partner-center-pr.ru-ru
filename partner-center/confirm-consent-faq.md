## <a href="" id="processfaq"></a>Процесс

**Каков новый процесс подтверждения принятия клиентом соглашения Microsoft Cloud?**

Корпорация Майкрософт считает своим долгом помогать своим клиентам и партнерами обеспечивать соответствие нормативным требованиям. Как партнер в программе поставщика облачных решений Майкрософт (CSP), вы несете ответственность за проверку положений и условий корпорации Майкрософт, включая использование продуктов и служб Майкрософт с клиентами. Чтобы помочь партнерам в принятии решений о соответствии требованиям и обеспечить прозрачность, мы создаем новый процесс, требующий партнеров CSP, чтобы убедиться, что их клиенты приняли условия соглашения Microsoft Cloud. По состоянию на 7 августа 2018 мы предоставили новое поле в пользовательском интерфейсе панели мониторинга центра партнеров, а также через API центра партнеров для предоставления этих сведений партнерам. Изначально она будет необязательной для подтверждения, но потребуется начать с 7 ноября 2018. 

**Когда это требование вступит в действие?**

Партнеры уже могут подтверждать принятие клиентами условий соглашения (функция стала доступна 7 августа 2018 года). Начиная с 7 ноября 2018, пользователи панели мониторинга центра партнеров должны подтвердить принятие Клиентом соглашения Microsoft Cloud для всех новых транзакций. 22 марта 2019. пользователи API центра партнеров должны будут выполнять те же действия. Крайний срок для пользователей API был расширен, чтобы предоставить им дополнительное время для технической реализации.

**СОЗДАТЬ | Проверяет ли корпорация Майкрософт точность сведений о клиенте, предоставленных партнером?**

Корпорация Майкрософт проверяет, что указанный адрес электронной почты клиента имеет правильный формат. Ответственность за предоставление точной информации и исправление всех неточных данных. 

**СОЗДАТЬ | Как партнер может доказать согласие клиента на Microsoft Cloud соглашение, является ли оно подписанным, документированным, оцифрованным или на бумаге?**

Партнер несет ответственность за гарантию того, что клиент принял условия соглашения Microsoft Cloud и для управления собственными процессами и записями. Используя панель мониторинга центра партнеров или процесс API центра партнеров, партнер подтверждает, что клиент принял условия соглашения Microsoft Cloud.

**СОЗДАТЬ | Как корпорация Майкрософт будет проверять, завершен ли процесс подтверждения для клиента?**

Начиная с 7 ноября 2018, когда партнер CSP инициирует транзакцию с помощью панели мониторинга центра партнеров для нового или существующего клиента, системы Майкрософт автоматически проверяют, есть ли сведения о подтверждении клиента в центре партнеров. Панели мониторинга. Если они отсутствуют, транзакция не будет обработана. 

Начиная с 22 марта 2019, когда партнер CSP инициирует транзакцию для нового или существующего клиента с помощью API центра партнеров, системы Майкрософт автоматически проверяют, есть ли сведения о подтверждении клиента. Если они отсутствуют, транзакция не будет обработана. 

**СОЗДАТЬ | Отправляется ли уведомление клиентам, чей адрес электронной почты указан в процессе подтверждения?**

Процесс уведомления отсутствует.

## <a href="" id="policyfaq"></a>Политик

**Почему корпорация Майкрософт требует партнеров подтвердить, что клиенты приняли Соглашение Microsoft Cloud?** 

Соглашение Microsoft Cloud Agreement защищает партнеров, клиентов и корпорацию Майкрософт, определяя права клиентов на использование и условия, касающиеся продуктов и служб Microsoft Cloud. 

Правильное выполнение соглашения Microsoft Cloud защищает наших партнеров, клиентов и Майкрософт, обеспечивая взаимное выравнивание по многим важным темам, включая, помимо прочего, безопасность, конфиденциальность и защиту данных. Процесс подтверждения обеспечивает прозрачность и выравнивание и позволяет корпорации Майкрософт более эффективно реагировать на запросы нормативных требований.

**Какие сведения о клиенте необходимы для завершения процесса подтверждения?**
Необходимо предоставить следующие сведения о пользователе, принявшем условия соглашения.
-   Дата принятия условий соглашения
-   Имя и фамилия
-   Адрес электронной почты
-   Номер телефона (необязательно) 

**СОЗДАТЬ | Что произойдет, если мы не будем соответствовать дате оплаты?**

Партнеры уже могут подтверждать принятие клиентами условий соглашения (функция стала доступна 7 августа 2018 года). Начиная с 7 ноября 2018, пользователи панели мониторинга центра партнеров должны подтвердить принятие Клиентом соглашения Microsoft Cloud для всех новых транзакций. 22 марта 2019. пользователи API центра партнеров должны будут выполнять те же действия.

Во время применения новые покупки для существующего клиента или изменения количества мест в существующих подписках не будут разрешены до подтверждения принятия клиентом. До подтверждения принятия условий соглашения нельзя будет совершать транзакции для новых клиентов. Это требование не затрагивает автоматическое продление подписок.

**К каким регионам применяется это требование?**

Подтверждение принятия клиентом соглашения Microsoft Cloud требуется для всех регионов в общедоступном облаке Майкрософт. Подтверждение принятия клиентом условий соглашения не поддерживается в национальных облаках (в том числе в Microsoft Cloud for US Government и Microsoft Cloud в Германии).

**Какие партнеры отвечают за подтверждение принятия клиентом соглашения Microsoft Cloud?**

Начиная с 7 ноября, 2018, непрямые поставщики (CSP) поставщика облачных служб и прямые партнеры по оплате CSP несут ответственность за подтверждение принятия клиентов и могут использовать API центра партнеров или панель мониторинга центра партнеров. Косвенным поставщикам может потребоваться сначала получить необходимые сведения от непрямых торговых посредников, а затем сделать их доступными для Майкрософт через API центра партнеров или на панели мониторинга центра партнеров. По договору, это непрямой поставщик с контрактом привязки к Майкрософт, гарантирующий, что их конечные клиенты-посредники CSP приняли Соглашение Microsoft Cloud. 

**Какие сценарии клиентов нуждаются в подтверждении принятия клиентом соглашения Microsoft Cloud?**

Принятие соглашения Microsoft Cloud должно быть подтверждено, когда партнеры CSP хотят выполнить следующие транзакции:
1.  Новый клиент завершает новые покупки
2.  Существующий клиент завершает новые покупки
3.  Существующий клиент обновляет число рабочих мест для существующих подписок

**Можно ли подтвердить принятие существующих клиентов отдельно от новых покупок или дополнений к области?**

Да. Принятие клиентов может быть подтверждено для всех клиентов, охваченных Microsoft Cloudным соглашением.

**Каковы требования для клиентов, имеющих несколько партнеров CSP?**

Принятие клиентом условий соглашения подтверждает партнер. Если клиент проводит сделки с несколькими партнерами CSP, каждый партнер должен будет подтвердить принятие клиентом условий соглашения.

**Я являюсь непрямым торговым посредником CSP, работающим с непрямыми поставщиками CSP. Что такое моя ответственность?**

По договору, это непрямой поставщик с контрактом привязки к Майкрософт, гарантирующий, что конечный клиент принял Microsoft Cloudное соглашение. Однако между косвенными поставщиками и косвенными торговыми посредниками могут быть отдельные договоренности в отношении сбора сведений о принятии клиентами условий соглашения. В этом случае косвенным поставщикам потребуется сначала получить необходимые сведения о принятии клиентами условий соглашения от их косвенных торговых посредников, а затем предоставить их корпорации Майкрософт через API Центра партнеров или панель мониторинга Центра партнеров. Вы можете помочь в проведении этой процедуры, быстро передавая необходимые сведения своим косвенным поставщикам по их запросу.

**Я являюсь непрямым поставщиком CSP. Нужно ли координировать непрямой торгового посредника CSP для подготовки?**

Косвенным поставщикам, которым требуется получить сведения о принятии клиентом условий соглашения от своих косвенных торговых посредников, рекомендуется незамедлительно сообщать об этой необходимости своим косвенным торговым посредникам, чтобы согласовать доставку и получение этой информации.

**Как можно объяснить своим клиентам, что они должны подтвердить принятие соглашения Microsoft Cloud?**

Правильное выполнение соглашения Microsoft Cloud защищает корпорации Майкрософт, ее партнеров и клиентов, обеспечивая взаимное выравнивание по многим важным темам, включая, помимо прочего, безопасность, конфиденциальность и защиту данных. Процесс подтверждения обеспечивает прозрачность и выравнивание и позволяет корпорации Майкрософт более эффективно реагировать на запросы нормативных требований.

**СОЗДАТЬ | Если клиент работает с одним непрямым поставщиком CSP, но транзакции выполняются с несколькими непрямыми торговыми посредниками CSP, предоставит ли каждый непрямому торговому посреднику CSP сведения о подтверждении?**

Непрямой поставщик CSP отвечает за предоставление подтверждения согласия клиента на Microsoft Cloud в корпорацию Майкрософт. Для каждой учетной записи клиента только одно подтверждение может быть записано одновременно на панели мониторинга центра партнеров. 

Непрямой поставщик CSP несет ответственность за подтверждение того, что клиентское лицо, ответственное за согласие с соглашением Microsoft Cloud, не учитывает число непрямых торговых посредников CSP, с которыми они выполнили инструкцию.

**СОЗДАТЬ | Кто должен быть полномочным лицом, чтобы подтвердить принятие соглашения Microsoft Cloud в клиентской организации?**

Ответственность за получение клиентом соглашения Microsoft Cloud от лица, входящего в организацию клиента, несет партнер. Это обязанность клиента определить, кто является полномочным для предоставления согласия. 

**СОЗДАТЬ | Какая версия соглашения Microsoft Cloud является партнером, отвечающим за подтверждение принятия клиентом данных? Должны ли партнеры указать, какая версия соглашения Microsoft Cloud была принята клиентом?**

Партнеры несут ответственность за использование наиболее актуального соглашения Microsoft Cloud при создании транзакций CSP с новым клиентом. Партнеры должны подтвердить принятие наиболее актуального соглашения Microsoft Cloud при продлении, изменении или завершении новых транзакций с существующим клиентом. 

В случае, если соглашение Microsoft Cloud обновляется до продления, партнерам не требуется, чтобы клиент повторно получал согласие Microsoft Cloudного соглашения, но сначала должен подтвердить согласие на Microsoft Cloud соглашение до для продления. Однако может потребоваться, чтобы клиент принимал новые условия до продления, чтобы воспользоваться преимуществами новых и усовершенствованных условий в соглашении Microsoft Cloud.

**СОЗДАТЬ | Я являюсь непрямым торговым посредником CSP. Если я занимаю условия соглашения Microsoft Cloud в записях моей компании, нужно ли также предоставлять эти сведения через панель мониторинга центра партнеров или через API центра партнеров?**

По договору, это непрямой поставщик с контрактом привязки к Майкрософт, гарантирующий, что конечный клиент принял Microsoft Cloudное соглашение. Однако между косвенными поставщиками и косвенными торговыми посредниками могут быть отдельные договоренности в отношении сбора сведений о принятии клиентами условий соглашения. В этом случае косвенным поставщикам потребуется сначала получить необходимые сведения о принятии клиентами условий соглашения от их косвенных торговых посредников, а затем предоставить их корпорации Майкрософт через API Центра партнеров или панель мониторинга Центра партнеров.

**СОЗДАТЬ | Что делать, если мой клиент не согласен с условиями соглашения Microsoft Cloud?**

В качестве партнера в программе CSP корпорации Майкрософт вы несете ответственность за принятие Клиентом соглашения о Microsoft Cloud до того, как вы помещаете заказ. Это не новое требование. По любой причине, если клиент больше не согласен с соглашением Microsoft Cloud, он должен отменить подписки, связанные с этим клиентом.

**СОЗДАТЬ | Может ли торговый посредник предоставлять согласие от имени клиента?**

Нет.

**Когда я подтверждаю, что клиент предоставил согласие, нужно ли повторно подтвердить его для клиента в будущем?**

После подтверждения того, что клиент принял Соглашение Microsoft Cloud, эти сведения остаются в разделе Учетная запись клиента в центре партнеров, и вам не нужно повторно подтверждать принятие в течение времени, которое охватывает клиент. условия соглашения. Перед размещением нового заказа необходимо только повторно подтвердить принятие.

В случае, если соглашение Microsoft Cloud обновляется до продления, партнерам не требуется, чтобы клиент повторно получал согласие Microsoft Cloudного соглашения, но сначала должен подтвердить согласие на Microsoft Cloud соглашение до для продления. Однако может потребоваться, чтобы клиент принимал новые условия до продления, чтобы воспользоваться преимуществами новых и усовершенствованных условий в соглашении Microsoft Cloud.

**СОЗДАТЬ | У меня нет ничего удобного с непосредственным контактным лицом корпорации Майкрософт. Как корпорация Майкрософт будет использовать эти сведения?**

Когда партнер создает клиент клиента, ему необходимо предоставить основную контактную информацию для клиента. В условиях использования веб-служб корпорация Майкрософт будет обрабатывать персональные данные только в соответствии с задокументированными инструкциями клиента.

## <a href="" id="technicalfaq"></a>Техническую

**Как партнеры подтверждают принятие клиентов и предоставляют его корпорации Майкрософт? Какие средства доступны для поддержки этого процесса?**

Существует два способа, начиная с 7 августа 2018 для оказания помощи партнерам: 
1.  Партнеры могут использовать API-интерфейс Центра партнеров, чтобы программным способом убедиться в принятии пользователем условий соглашения. 
2.  Партнеры могут использовать панель мониторинга центра партнеров для подтверждения принятия клиентом данных.

**СОЗДАТЬ | Я являюсь партнером CSP, использующим API центра партнеров. Какие изменения API необходимо учитывать?** 

В этом случае API изменится на учетную запись:

-   [https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/partner-center/develop/confirm-customer-consent)

**Что делать, если у меня возникли трудности при реализации API центра партнеров?**

Принятие клиентом условий соглашения можно подтвердить вручную через панель мониторинга Центра партнеров.

**СОЗДАТЬ | Вместо того, чтобы использовать панель мониторинга центра партнеров или API центра партнеров, можно ли выполнить операцию по загрузке файла с информацией о подтверждении клиента?**

В настоящее время панель мониторинга центра партнеров и API центра партнеров являются единственным приемлемым средством предоставления подтверждения принятия клиентом соглашения о Microsoft Cloud.

**СОЗДАТЬ | Я партнер, использующий платформу стороннего производителя или "по промежуточного слоя", который выполняет функции подключения систем моей компании к центру партнеров. Что мне нужно сделать?**

Для партнеров, использующих платформы, которые интегрируют API центра партнеров, рекомендуется напрямую говорить с поставщиками о планах и временных шкалах для интеграции функций API центра партнеров, которые автоматизируют процесс подтверждения клиента.

**СОЗДАТЬ | Можно ли предоставить пример потока API для импорта полного имени клиента, адреса электронной почты и даты Microsoft Cloud принятия соглашений?**

Партнерам рекомендуется ознакомиться с документацией по API центра партнеров, чтобы получить дополнительные сведения и примеры.

**СОЗДАТЬ | Разделы справки получить самое актуальное соглашение Microsoft Cloud, которое будет отображаться для клиента и использовать в записи подтверждения как часть процесса API?**

Текущий API центра партнеров является очень базовым и не предоставляет партнерам программно получать копию Microsoft Cloud соглашения, а также не собирает версии соглашения Microsoft Cloud, принятых клиентом в ходе подтверждения. 

**СОЗДАТЬ | Для использования API центра партнеров для этого процесса требуется параметр userID. Разделы справки управлять этим требованием, когда заказы размещаются через сторонние платформы, которые не интегрируются с Azure AD?**

В этом случае userID определяет сотрудника в партнерской клиенте, который предоставляет подтверждение согласия клиента. Если у сотрудника нет соответствующей учетной записи пользователя Azure AD в клиенте партнера, вам будет предложено назначить конкретную учетную запись пользователя для этой цели и создать процесс для отдельной записи удостоверения сотрудника, предоставляющего операций. 

**СОЗДАТЬ | Почему корпорация Майкрософт требует от партнера подтвердить согласие клиента на Microsoft Cloudное соглашение, а не получать его непосредственно от клиента?**

Клиент приобретается у партнера CSP, а не непосредственно в корпорации Майкрософт. Для использования служб, предоставляемых корпорацией Майкрософт, корпорация Майкрософт предоставляет термины, описанные в соглашении Microsoft Cloud. Для соглашения о торговом посреднике Microsoft Cloud требуются партнеры CSP, чтобы клиенты могли проверить и принять соглашение Microsoft Cloud до того, как будет размещен заказ. 

**СОЗДАТЬ | Я использую API и веб-интерфейс панели мониторинга центра партнеров для подтверждения Microsoft Cloudного соглашения о принятии клиентов. После 7 ноября 2018, какие варианты подтверждают принятие клиентом?**

Оба варианта доступны и приемлемы для подтверждения Microsoft Cloud принятия клиентом соглашения. Для транзакций, использующих панель мониторинга центра партнеров, потребуется подтверждение, которое должно быть установлено начиная с 7 ноября 2018. Для транзакций, использующих API центра партнеров, будет требоваться подтверждение, начиная с 22 марта 2019.
