## <a href="" id="processfaq"></a>Процесс

**Что такое нового процесса для подтверждения того, условия приемки клиентом соглашения Microsoft Cloud**

Корпорация Майкрософт считает своим долгом помогать своим клиентам и партнерами обеспечивать соответствие нормативным требованиям. Как партнер в программе поставщика облачных решений (CSP) корпорации Майкрософт вы несете ответственность за рецензирования Microsoft условиями, включая приемлемое использование продуктов и служб с клиентами Майкрософт. Чтобы лучше помочь партнерам в соответствии нормативным требованиям и обеспечить прозрачность, мы поощряем новый процесс, требующий партнеров CSP, чтобы убедиться, что клиентам приняли условия соглашения для Microsoft Cloud. Начиная с 7 августа 2018 года мы ввели новое поле в панели мониторинга центра партнеров пользовательском интерфейсе также API центра партнеров для партнеров для предоставления этих сведений. Изначально он будет необязательным для предоставления подтверждения, однако он потребуется начиная с 7 ноября 2018 г. 

**Если это требование вступают в силу?**

Партнеры уже могут подтверждать принятие клиентами условий соглашения (функция стала доступна 7 августа 2018 года). Начиная с 7 ноября 2018 г панели мониторинга центра партнеров пользователям требуется подтвердить условия приемки клиентом соглашения Microsoft Cloud на всех новых транзакций. На 22 марта 2019 API центра партнеров пользователи должны будут делать то же самое. Крайний срок для пользователей API была расширена для них дополнительные времени для технической реализации.

**НОВЫЕ | Проверка выполняется точность информации клиента, предоставленный партнером Майкрософт?**

Майкрософт проверяет, что указанный адрес электронной почты клиента имеет неправильный формат. Отвечает за партнера предоставление точной информации и исправьте все сведения, является неточным. 

**НОВЫЕ | Как партнер порою Приемка соглашения Microsoft Cloud, подписан ли он, задокументированы, Оцифрованные, или на бумаге?**

Партнер отвечает за то, что клиент принял условия соглашения для Microsoft Cloud, так и для управления свои собственные процесса и записи. С помощью процесса панели мониторинга центра партнеров или API центра партнеров, партнер подтверждение, что клиент принял условия соглашения для Microsoft Cloud.

**НОВЫЕ | Как корпорация Майкрософт проверит, процесс подтверждения завершения для клиента?**

Начиная с ноября 7 2018 г., когда партнер CSP инициирует транзакцию с помощью панели мониторинга центра партнеров для нового или существующего клиента, системы Майкрософт автоматически проверяет, присутствует ли сведения о подтверждении клиента в центре партнеров Панель мониторинга. Если они отсутствуют, транзакция не будет обработана. 

Начиная с 22 марта 2019 г., когда партнер CSP инициирует транзакцию для нового или существующего клиента с помощью API центра партнеров, систем корпорации Майкрософт автоматически проверит, присутствует ли сведения о подтверждении клиента. Если они отсутствуют, транзакция не будет обработана. 

**НОВЫЕ | Уведомление отправляется клиентам, адрес электронной почты которых предоставляется в процессе подтверждения?**

Отсутствует процесс уведомления.

## <a href="" id="policyfaq"></a>Политики

**Почему Майкрософт требует включить партнеров, чтобы убедиться, что клиенты приняли соглашение об облаке?** 

Соглашение об облаке защищает партнерами, клиентами и Майкрософт, предоставляя права на использование клиента, правила и условия, предоставленные для заказов продуктов и служб Microsoft Cloud. 

Соглашение об облаке правильного выполнения защищает наших партнеров, клиентов и Майкрософт, обеспечивая взаимной выравнивание по много важным темам, включая, но не ограничиваясь безопасности, конфиденциальности и защиты данных. Процесс подтверждения обеспечивает прозрачность и выравнивание и позволяет лучше помочь клиентам, которым требуется своевременно отвечать на запросы нормативным.

**Какие сведения клиента необходим для завершения процесса подтверждения?**
Необходимо предоставить следующие сведения о пользователе, принявшем условия соглашения.
-   Дата принятия условий соглашения
-   Имя и фамилия
-   Адрес электронной почты
-   Номер телефона (необязательно) 

**НОВЫЕ | Что произойдет, если мы не соответствуют срок?**

Партнеры уже могут подтверждать принятие клиентами условий соглашения (функция стала доступна 7 августа 2018 года). Начиная с 7 ноября 2018 г панели мониторинга центра партнеров пользователям требуется подтвердить условия приемки клиентом соглашения Microsoft Cloud на всех новых транзакций. На 22 марта 2019 API центра партнеров пользователи должны будут делать то же самое.

Во время принудительного применения новых покупках для существующего клиента или изменения количества на существующие подписки рабочих мест не разрешается до подтверждения условия приемки клиентом. До подтверждения принятия условий соглашения нельзя будет совершать транзакции для новых клиентов. Это требование не затрагивает автоматическое продление подписок.

**Какие регионы это требование относится к?**

Подтверждение условия приемки клиентом, чтобы соглашение об облаке является обязательным для всех регионов в общедоступное облако Microsoft. Подтверждение принятия клиентом условий соглашения не поддерживается в национальных облаках (в том числе в Microsoft Cloud for US Government и Microsoft Cloud в Германии).

**Каким партнерам несут ответственность за подтверждение условия приемки клиентом в соглашение Microsoft Cloud?**

Начиная с 7 ноября 2018 г., непрямые поставщики поставщика облачных служб (CSP) и прямой счета CSP партнеры ответственны за подтверждение условия приемки клиентом и может использовать API центра партнеров или панели мониторинга центра партнеров для этого. Непрямые поставщики может потребоваться сначала собрать необходимые сведения из своих непрямых торговых посредников, а затем сделать его доступным в корпорацию Майкрософт с помощью API центра партнеров или панели мониторинга центра партнеров. В договорах это непрямой поставщик, у кого есть привязки контракта с корпорацией Майкрософт, чтобы убедиться, что их торговые посредники CSP конечным пользователям принятия соглашения Microsoft Cloud. 

**Какие сценариях клиента требуется подтверждение условия приемки клиентом соглашения облака Microsoft?**

Необходимо подтвердить принятие соглашение об облаке, при партнеры CSP должны быть следующие транзакции:
1.  Новый клиент завершает новых покупках
2.  Существующий клиент завершает новых покупках
3.  Существующий клиент обновляет количество рабочих мест для существующих подписок

**Можно подтвердить принятие для существующих клиентов, отдельно от их новых покупках или рабочих мест дополнения?**

Да. Условия приемки клиентом, может быть подтверждена для всех клиентов, распространяется соглашение об облаке.

**Каковы требования для клиентов, имеющих несколько партнеров CSP?**

Принятие клиентом условий соглашения подтверждает партнер. Если клиент проводит сделки с несколькими партнерами CSP, каждый партнер должен будет подтвердить принятие клиентом условий соглашения.

**Я непрямым торговым посредником CSP работа с непрямыми поставщиками CSP. Что такое моей ответственности?**

В договорах это непрямой поставщик, у кого есть привязки контракта с корпорацией Майкрософт, чтобы убедиться, что конечный пользователь принял соглашение Microsoft Cloud. Однако между косвенными поставщиками и косвенными торговыми посредниками могут быть отдельные договоренности в отношении сбора сведений о принятии клиентами условий соглашения. В этом случае косвенным поставщикам потребуется сначала получить необходимые сведения о принятии клиентами условий соглашения от их косвенных торговых посредников, а затем предоставить их корпорации Майкрософт через API Центра партнеров или панель мониторинга Центра партнеров. Вы можете помочь в проведении этой процедуры, быстро передавая необходимые сведения своим косвенным поставщикам по их запросу.

**Я непрямой поставщик CSP. Требуется для координации с моей непрямым торговым посредником CSP для подготовки?**

Косвенным поставщикам, которым требуется получить сведения о принятии клиентом условий соглашения от своих косвенных торговых посредников, рекомендуется незамедлительно сообщать об этой необходимости своим косвенным торговым посредникам, чтобы согласовать доставку и получение этой информации.

**Как я, объясняет моих клиентов, их следует убедиться в их принятия соглашения облака Microsoft?**

Соглашение об облаке правильного выполнения защищает Microsoft, ее партнеров и клиентов, обеспечивая взаимной выравнивание по много важным темам, включая, но не ограничиваясь безопасности, конфиденциальности и защиты данных. Процесс подтверждения обеспечивает прозрачность и выравнивание и позволяет лучше помочь клиентам, которым требуется своевременно отвечать на запросы нормативным.

**НОВЫЕ | Если клиент работает с одной непрямой поставщик CSP, но транзакции во время завершения нескольких непрямые торговые посредники CSP, каждый непрямым торговым посредником CSP требуется ли предоставлять непрямые поставщики CSP с сведения о подтверждении?**

Непрямые поставщики CSP отвечает за предоставление подтверждение условия приемки клиентом соглашение по использованию Microsoft Cloud в корпорацию Майкрософт. Для каждой учетной записи клиента только одно подтверждение может быть записано в панели мониторинга центра партнеров за один раз. 

Непрямые поставщики CSP несет ответственность за подтверждение, что ответственность за согласие на соглашение об облаке клиента контактное лицо может это сделать, независимо от количества непрямые торговые посредники CSP, который они transact с.

**НОВЫЕ | Тех, кто будет уполномоченное лицо, чтобы подтвердить принятие соглашения Microsoft Cloud в организации клиента?**

Отвечает партнера для получения условия приемки клиентом соглашения облака Майкрософт от лица в организации клиента. Это обязанность для определения прав для предоставления приемки. 

**НОВЫЕ | Какая версия соглашение об облаке являются партнерами, ответственность за подтверждение условия приемки клиентом из? Необходимо указать, какая версия соглашение об облаке был принят клиентом партнеров?**

Партнеры несут ответственность за использование наиболее текущего соглашения Microsoft Cloud, при выполнении транзакций CSP с помощью нового клиента. Партнерам необходимо подтвердить принятие самые последние облака Майкрософт соглашения при продление, изменение или завершение работы новых операций с существующим клиентом. 

В случае, что соглашение об облаке обновляется до обновления не обязательно должны иметь клиент немедленно повторно получить условия приемки клиентом соглашение облачных партнеров, но должны повторно подтвердить согласие на предварительного соглашение по использованию Microsoft Cloud для обновления. Тем не менее вы можете принять новые условия до обновления, чтобы воспользоваться преимуществами новых и улучшенных условия соглашения для Microsoft Cloud на заказчика.

**НОВЫЕ | Я непрямым торговым посредником CSP. Если в моей компании записей записи условия приемки клиентом соглашения Microsoft Cloud, нужно также предоставить эти сведения с помощью панели мониторинга центра партнеров или API центра партнеров?**

В договорах это непрямой поставщик, у кого есть привязки контракта с корпорацией Майкрософт, чтобы убедиться, что конечный пользователь принял соглашение Microsoft Cloud. Однако между косвенными поставщиками и косвенными торговыми посредниками могут быть отдельные договоренности в отношении сбора сведений о принятии клиентами условий соглашения. В этом случае косвенным поставщикам потребуется сначала получить необходимые сведения о принятии клиентами условий соглашения от их косвенных торговых посредников, а затем предоставить их корпорации Майкрософт через API Центра партнеров или панель мониторинга Центра партнеров.

**НОВЫЕ | Что делать, если мой клиент не согласны с условиями соглашения для облака Microsoft?**

Как партнер в программе CSP вы несете ответственность за Приемка клиентом соглашения Microsoft Cloud до размещения заказа. Это новое требование. По любой причине если клиент не соглашается с соглашением Microsoft Cloud, партнер должен отменить подписки, связанные с этим клиентом.

**НОВЫЕ | Торгового посредника, может предоставить согласие от имени клиента?**

Нет.

**Когда я подтверждаю, что клиент предоставил приемки, нужно ли повторно подтвердить его для клиента в будущем?**

После подтверждения того, что клиент принял соглашение Microsoft Cloud, они остаются в разделе учетной записи клиента в центре партнеров и нет необходимости повторно подтвердить принятие продолжительность времени, которая рассматривается клиента условия соглашения. Необходимо только подтверждения принятия перед осуществлением нового заказа.

В случае, что соглашение об облаке обновляется до обновления не обязательно должны иметь клиент немедленно повторно получить условия приемки клиентом соглашение облачных партнеров, но должны повторно подтвердить согласие на предварительного соглашение по использованию Microsoft Cloud для обновления. **НОВЫЕ | Я не удовлетворен предоставляя Microsoft my конечному клиенту прямой контактные данные. Как Майкрософт использует эти сведения?**

Когда партнер создает с клиентом пользователя, это необходимо, чтобы предоставить основные контактные данные для клиента. В условиях обслуживания «Корпорация Майкрософт будет обрабатывать персональные данные только на приведенные инструкции от клиента.»

## <a href="" id="technicalfaq"></a>Технические

**Как партнеры Подтвердите условия приемки клиентом и передавать его в корпорацию Майкрософт? Какие средства доступны для поддержки данного процесса?**

Существует два метода, которые доступны начиная с 7 августа 2018 г. для партнеров: 
1.  Партнеры могут использовать API-интерфейс Центра партнеров, чтобы программным способом убедиться в принятии пользователем условий соглашения. 
2.  Партнеры могут использовать панели мониторинга центра партнеров, чтобы подтвердить условия приемки клиентом.

**НОВЫЕ | Я являюсь партнером CSP, который использует API центра партнеров. Какие изменения API нужно учитывать?** 

В этом случае изменения API, чтобы учесть являются:

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**Что делать, если не получается, реализация API центра партнеров?**

Принятие клиентом условий соглашения можно подтвердить вручную через панель мониторинга Центра партнеров.

**НОВЫЕ | В качестве альтернативы с помощью панели мониторинга центра партнеров или API центра партнеров можно ли выполнять массовое отправьте файл с информацией о клиентах подтверждение?**

В настоящее время панели мониторинга центра партнеров и API центра партнеров — принятые исключительную предоставления подтверждение условия приемки клиентом соглашения Microsoft Cloud.

**НОВЫЕ | Я являюсь партнером, с помощью сторонних платформ, или «промежуточные», который выполняет функции для подключения систем моей компании с помощью центра партнеров. Что нужно сделать?**

Вы сможете поговорить с поставщиков относительно их планы и временных шкал для интеграции функций API центра партнеров, который автоматизирует процесс подтверждения клиента рекомендуется партнерами с помощью платформы, интегрирующие API центра партнеров.

**НОВЫЕ | Могли бы вы привести пример потока API для импорта полное имя клиента, адрес электронной почты и Дата принятия соглашения Microsoft Cloud?**

Партнеры рекомендуется в документации по API центра партнеров Дополнительные сведения и примеры.

**НОВЫЕ | Как получить самые последние соглашения Microsoft Cloud для отображения для клиента и использовать в записи подтверждение как часть процесса Мои API?**

Текущий API центра партнеров очень прост и не предоставляет для партнеров, чтобы программно получить копию соглашения Microsoft Cloud, а также записи версий соглашения Microsoft Cloud, принимается клиентом как часть подтверждения. 

**НОВЫЕ | С помощью API центра партнеров для этого процесса требуется параметр userID. Как управлять это требование, когда заказы оформляются с помощью сторонних платформ, которые не интегрируются с Azure AD?**

Идентификатор пользователя в данном случае идентифицирует служащего в разделе клиента-партнера, предоставляющими подтверждения согласия клиента. Если сотрудник не имеет соответствующую учетную запись пользователя Azure AD в клиенте партнера, то рекомендуется назначить для этой цели специальную учетную запись и создать процесс для отдельно захвата идентификатор сотрудника, предоставляя Подтверждение. 

**НОВЫЕ | Почему Майкрософт требует включить для подтверждения согласия клиента для соглашения Microsoft Cloud вместо получения его непосредственно из клиента партнера?**

Клиент приобретает от партнера CSP, не непосредственно от корпорации Майкрософт. Для использования служб, предоставляемых Майкрософт, корпорация Майкрософт предоставляет термины, описанных в соглашении Microsoft Cloud. Соглашения торгового посредника Microsoft Cloud требует партнеров CSP для клиентов, просмотрите и примите условия соглашения Microsoft Cloud до размещения заказа. 

**НОВЫЕ | Я использую веб-интерфейсом API и панели мониторинга центра партнеров, чтобы подтвердить условия приемки клиентом соглашение по использованию Microsoft Cloud. После 7 ноября 2018 г. какие варианты доступны для подтверждения условия приемки клиентом?**

Оба варианта доступны приемлемо для подтверждения того, условия приемки клиентом соглашение по использованию Microsoft Cloud. Транзакции с использованием панели мониторинга центра партнеров потребует подтверждения должны присутствовать, начиная с 7 ноября 2018 г. Транзакции, использующие API центра партнеров потребует подтверждения должны присутствовать, начиная от 22 марта 2019 г.
