## <a href="" id="processfaq"></a>Process

**Что такое новый процесс для подтверждения того, клиент принятие соглашения Microsoft Cloud?**

Корпорация Майкрософт стремится помогает своим клиентам и партнерам сохраните соответствие нормативным требованиям. Как партнер в программе поставщиков облачных решений (CSP) Майкрософт вы несете ответственность за проверку Майкрософт положений и условий, включая допустимого использования продуктов и служб с клиентами Майкрософт. Лучше помочь партнерам в обеспечении соответствия нормативным требованиям и предотвращает прозрачность, мы включается новый процесс, требующий партнеры CSP, чтобы убедиться, что своим клиентам приняли условия соглашения Microsoft Cloud. Начиная с 7 августа 2018 г. мы разработали новые поля в информационной панели центра партнеров пользовательского интерфейса также API центра партнеров для партнерам для предоставления этих сведений. Изначально их необязательно предоставлять подтверждения, но он потребуется 7 ноября 2018 г. 

**Если это требование вступают в силу?**

Партнеры могут начать подтверждение принятия сразу же функциональность доступна 7 августа 2018 г. Начиная с 7 ноября 2018 г., информационной панели центра партнеров от пользователей требуется для подтверждения клиента принятие соглашения Microsoft Cloud для всех новых операций. 22 марта 2019 года API центра партнеров пользователям потребуется сделать то же самое. Крайний срок для пользователей API была расширена для них дополнительные время технические внедрения.

**НОВЫЕ | Проверка Майкрософт точность сведений клиента, полученное от партнера?**

Майкрософт проверяет, что указанный адрес электронной почты клиента имеет правильный формат. Обязано партнера предоставление точной информации и информации, которая является неточным. 

**НОВЫЕ | Как можно партнера подтвердить согласие клиента соглашения Microsoft Cloud, ли он подписан, описаны, это, или на бумаге?**

Партнер ответственность за соответствие, что пользователь принял условия соглашения Microsoft Cloud для управления своими процесс и записи. С помощью информационной панели центра партнеров или API центра партнеров процесс, партнер подтверждения того, что пользователь принял условия соглашения Microsoft Cloud.

**НОВЫЕ | Как корпорация Майкрософт проверит, подтверждения процесс будет завершен для клиента?**

Начиная 7 ноября 2018 г., когда партнер CSP инициирует транзакцию с помощью информационной панели центра партнеров для новых или существующих клиентов, систем Майкрософт автоматически проверит сведения о подтверждении клиент при наличии в центре партнеров Панель мониторинга. Если его нет, транзакции не смогут быть завершена. 

Начиная с 22 марта 2019 г., когда партнер CSP инициирует транзакции для новых или существующих клиентов с помощью API центра партнеров, систем Майкрософт автоматически проверит при наличии подтверждения сведений о клиенте. Если его нет, транзакции не смогут быть завершена. 

**НОВЫЕ | Уведомление, отправленное клиентам, чьи адреса электронной почты предоставляется в процессе подтверждения?**

Нет ни один процесс уведомления.

## <a href="" id="policyfaq"></a>Политика

**Почему Microsoft требует партнеров, чтобы убедиться, что пользователи приняли соглашения Microsoft Cloud?** 

Microsoft Cloud Agreement защищает партнерами, клиентами и Microsoft, предоставляя права на использование клиента и условия, предоставленные для Microsoft Cloud заказов продуктов и служб. 

Правильного выполнения соглашения Microsoft Cloud защищает наших партнеров, клиентов и Майкрософт за счет взаимной выравнивание много важных тем в том числе, но не ограничиваясь безопасности, конфиденциальности и защиты данных. Процесс подтверждения прозрачность и выравнивания и позволяет Майкрософт, чтобы лучше помочь клиентам, которым необходимо незамедлительно реагировать на запросы нормативным.

**Какие сведения о клиенте необходимо завершить процесс подтверждения?**
Следующие сведения является обязательным в отношении лица, приняли соглашение:
-   Дата принятия соглашения
-   Имени и фамилии
-   адрес электронной почты;
-   Номер телефона (необязательно) 

**НОВЫЕ | Что произойдет, если мы не соответствуют срок?**

Партнеры могут начать подтверждение принятия сразу же функциональность доступна 7 августа 2018 г. Начиная с 7 ноября 2018 г., информационной панели центра партнеров от пользователей требуется для подтверждения клиента принятие соглашения Microsoft Cloud для всех новых операций. 22 марта 2019 года API центра партнеров пользователям потребуется сделать то же самое.

Во время применения новых покупок для существующего клиента или изменения число количество на существующие подписки не разрешается до принятия клиента подтвержден. Для нового клиента транзакции не разрешается до принятия подтвержден. Это требование не будет снижена автоматическое продление.

**Области, которые это требование относится к?**

Подтверждение принятия клиента для Microsoft Cloud Agreement является обязательным для всех регионов в общедоступном облаке Microsoft. Подтверждение принятия клиента в национальных облаках (включая Microsoft Cloud для правительства США и Microsoft Cloud в Германии) не поддерживается.

**Какие партнеры несут ответственность за подтверждение принятия клиента для Microsoft Cloud Agreement?**

Вступления в силу 7 ноября 2018 г. поставщика облачных услуг (CSP) для косвенных поставщиков и прямой CSP выставления счетов партнерам обе ответственность за подтверждение принятия клиента и могут использовать API центра партнеров или информационной панели центра партнеров для этого. Непрямые поставщики может потребоваться сначала собирать необходимую информацию от своих торговых посредников и затем сделать его доступным в корпорацию Майкрософт через API центра партнеров или информационной панели центра партнеров. От лица это косвенного поставщика, который имеет привязку контракта с корпорацией Майкрософт, чтобы убедиться, что для Microsoft Cloud Agreement приняли торговых посредников CSP конечным пользователям. 

**Какие сценарии клиента требуют подтверждения принятия клиента Microsoft Cloud Agreement?**

Принятие соглашения Microsoft Cloud необходимо подтвердить при партнерам CSP требуется сделать следующие операции:
1.  Нового клиента завершается новых покупок
2.  Существующего клиента завершается новых покупок
3.  Обновление существующего клиента количество рабочих мест в существующие подписки

**Можно ли подтверждать принятие существующих клиентов, отдельно от своих новых покупок или число добавления?**

Да. Для всех пользователей, распространяется соглашения Microsoft Cloud расположение можно проверить принятия клиента.

**Каковы требования для клиентов, имеющих несколько партнеров CSP?**

Партнер подтверждено подтверждения соответствия. Если клиент занимаются с несколькими партнерами CSP, каждого партнера должен будет подтвердить принятия.

**Я являюсь работа с непрямыми поставщиками CSP торгового посредника CSP. Что такое Мои ответственности?**

От лица это косвенным поставщиком, имеющий контракта с корпорацией Майкрософт, чтобы убедиться, что конечному клиенту принял соглашения Microsoft Cloud привязки. Непрямые поставщики могут предусмотрено отдельных схем положений с непрямыми торговыми посредниками собирать принятия клиента. В этом случае косвенным поставщикам необходимо сначала собрать сведения принятия нужного клиента из своих торговых посредников и затем сделать его доступным в корпорацию Майкрософт через API центра партнеров или информационной панели центра партнеров. Этот процесс может поддерживать незамедлительно благодаря необходимую информацию для вашего косвенных поставщиков по их запросу.

**Я являюсь непрямым поставщиком CSP. Нужно ли координат с моих торгового посредника CSP для подготовки?**

Непрямые поставщики, которые необходимо собрать сведения принятия нужного клиента из своих торговых посредников рекомендуется для связи этого требования для своих торговых посредников немедленно для координации доставки и получения этой информации.

**Как можно ли объяснить моих клиентов, они должны проверить их принятие соглашения Microsoft Cloud?**

Правильного выполнения соглашения Microsoft Cloud защищает Microsoft, ее партнеров и клиентов за счет взаимной выравнивание много важных тем в том числе, но не ограничиваясь безопасности, конфиденциальности и защиты данных. Процесс подтверждения прозрачность и выравнивания и позволяет Майкрософт, чтобы лучше помочь клиентам, которым необходимо незамедлительно реагировать на запросы нормативным.

**НОВЫЕ | Если клиент работает с одной непрямым поставщиком CSP, но транзакции выполняются с помощью нескольких CSP торговых посредников, каждого торгового посредника CSP требуется предоставлять непрямым поставщиком CSP с информацией подтверждения?**

Косвенный поставщик CSP отвечает за подтверждение принятия клиента соглашения Microsoft Cloud в корпорацию Майкрософт. Для каждой учетной записи клиента только один подтверждения могут быть записаны в информационной панели центра партнеров за один раз. 

Для подтверждения того, что клиент контактное лицо ответственность за согласие соглашения Microsoft Cloud сделано таким образом, независимо от номера CSP торговых посредников, которым они взаимодействуете отвечает косвенного поставщика CSP.

**НОВЫЕ | Кто должен быть авторизованный пользователь, чтобы подтвердить принятие соглашения Microsoft Cloud внутри организации клиента?**

Обязано партнера для получения клиента принятие соглашения Microsoft Cloud от лица внутри организации клиента. Клиента обязано определить, кто авторизован для предоставления принятия. 

**НОВЫЕ | Какую версию Microsoft Cloud Agreement являются партнерами ответственность для подтверждения того, сообщающих о принятии клиентов? Партнеры нужно указать, какую версию Microsoft Cloud Agreement принят пользователем?**

Партнеры несут ответственность за использование самой соглашения Microsoft Cloud при создании транзакциям CSP с помощью нового клиента. Партнерам необходимо подтвердить принятие самой соглашения Microsoft Cloud при обновлении, изменение или завершение работы новых операций с существующим клиентом. 

В тех случаях, что Microsoft Cloud Agreement обновляется до обновления не должны иметь клиент немедленно повторно получить клиента принятие соглашения Microsoft Cloud партнерам, однако необходимо повторно подтвердить согласие на предварительного соглашения Microsoft Cloud для обновления. Тем не менее вы можете принять новые термины до обновления, чтобы воспользоваться преимуществами новых и улучшенных условия соглашения Microsoft Cloud клиенту.

**НОВЫЕ | Я являюсь торгового посредника CSP. Если я запись клиента принятие соглашения Microsoft Cloud в моей компании записей, нужно ли также предоставить эту информацию через информационную панель центра партнеров или API центра партнеров?**

От лица это косвенным поставщиком, имеющий контракта с корпорацией Майкрософт, чтобы убедиться, что конечному клиенту принял соглашения Microsoft Cloud привязки. В этом случае косвенным поставщикам необходимо сначала собрать сведения принятия нужного клиента из своих торговых посредников и затем сделать его доступным в корпорацию Майкрософт через API центра партнеров или информационной панели центра партнеров.

**НОВЫЕ | Что делать, если мой клиент не принимаю условия соглашения Microsoft Cloud?**

Как партнер в программе CSP корпорации Майкрософт вы несете ответственность за принятие клиента Microsoft Cloud Agreement до размещения заказа. Это не новое требование. По любой причине с согласия клиента больше не соглашение Microsoft Cloud, партнер должен отменить подписок, связанных с этого клиента.

**НОВЫЕ | Можно торгового посредника предоставить согласие от имени клиента?**

Нет.

**После убедитесь, что пользователь предоставил принятия, нужно ли повторно подтверждения клиента в будущем?**

После подтверждения клиент принял Microsoft Cloud Agreement, эта информация сохранится в разделе учетная запись клиента в центре партнеров и нет необходимости повторно подтверждая принятия в течение всего времени, распространяется клиента условия соглашения. Только должен повторно подтвердить принятия перед их размещением новый заказ.

В тех случаях, что Microsoft Cloud Agreement обновляется до обновления не должны иметь клиент немедленно повторно получить клиента принятие соглашения Microsoft Cloud партнерам, однако необходимо повторно подтвердить согласие на предварительного соглашения Microsoft Cloud для обновления. Когда партнер создает клиенте заказчика, она необходима для предоставления основное контактное лицо для клиента. Лицензионное веб-служб «Корпорация Майкрософт будет обрабатывают личные данные только на приведенные инструкции от клиента.»

## <a href="" id="technicalfaq"></a>Технические

Существуют два метода, которые будут доступны начиная с 7 августа 2018 г. Чтобы помочь партнерам: 
1.  Партнеры могут использовать API центра партнеров, чтобы программным образом подтвердить принятия клиента. 
2.  Партнеры могут использовать информационную панель центра партнеров для подтверждения принятия клиента.

**НОВЫЕ | Я являюсь партнером CSP, который использует API центра партнеров. Какие изменения API необходимо учитывать?**

Принятие расположение можно проверить вручную через информационную панель центра партнеров.

**НОВЫЕ | В качестве альтернативы с помощью информационной панели центра партнеров или API центра партнеров можно ли массовая передача файла с подтверждения сведений о клиенте вместо?**

В настоящее время информационной панели центра партнеров и API центра партнеров применяются только допустимых предоставления подтверждения принятия клиента Microsoft Cloud Agreement.

**НОВЫЕ | Я являюсь партнером, с помощью платформы третьих лиц, или «по промежуточного слоя,», выполняющий функции для подключения систем моей компании с помощью центра партнеров. Что нужно сделать?**

С помощью платформ, которые позволяют интегрировать API центра партнеров партнеры рекомендуется говорите непосредственно с поставщиками о своих планах и сроками для интеграции функций API центра партнеров, автоматизирует процесс подтверждения клиента.

**НОВЫЕ | Можно предоставить пример потока API для импорта клиента полное имя, адрес электронной почты и Дата принятия Microsoft Cloud Agreement?**

Партнеры рекомендуется просмотреть документацию API центра партнеров Дополнительные сведения и примеры.

**НОВЫЕ | Как получить наиболее актуальная соглашения Microsoft Cloud для отображения клиенту и использовать в записи подтверждения в рамках процесса Мои API?**

Текущий API центра партнеров очень простой и не предоставляет для партнеров, чтобы программным образом получить копию Microsoft Cloud Agreement, а также записи версиях Microsoft Cloud Agreement, принимаемый клиента в рамках подтверждения. 

**НОВЫЕ | С помощью API центра партнеров для этого процесса требует такого параметра userID. Как управлять этим требованиям, помещением заказов через платформ сторонних поставщиков, не интегрируется с Azure AD?**

В этом случае userID идентифицирует сотрудника в рамках клиента партнера, который предоставляет подтверждения согласия клиента. Если сотрудник не имеет соответствующий учетной записи пользователя Azure AD в рамках клиента партнера, то рекомендуется назначить учетной записи пользователя для этой цели и создания процесса по отдельности записать удостоверение сотрудника, предоставляя Подтверждение. 

**НОВЫЕ | Почему Microsoft требует партнер для подтверждения согласия клиента для Microsoft Cloud Agreement вместо получить его непосредственно из клиента?**
