## <a name="partner-contribution-indicators"></a>Индикаторы участия партнера

Индикаторы участия в партнерах (PCI) — это набор комплексных мер производительности, возможностей и успешности клиентов в Dynamics 365.

## <a name="scoring"></a>Способ

PCI использует шкалу в 100 точек, чтобы указать конечную 12-месячную выпускку успеха с помощью Dynamics 365 или Dynamics 365 Finance и Operations. Ниже приведены максимальные моменты, которые можно заработать для каждой из семи мер в PCI.

![Способ](images/pci1.png)

### <a name="thresholds"></a>Пороговые значения

Каждая метрика PCI имеет пороговые значения Майкрософт, которые могут заработать максимальные баллы компании. Эти пороговые значения отличаются для сотрудничества с клиентами и финансового & операций. Пороговые значения также отличаются для партнеров, работающих на развитых рынках, и для тех, кто работает исключительно на новых рынках. Если у партнера есть один или несколько клиентов на разработанном рынке, применяются пороговое значение на рынке. Чтобы применить новые пороговые значения для рынка, партнер должен иметь всех своих клиентов на новых рынках. (Разработанная или развивающаяся классификация рынка основана на определениях Microsoft Partner Network здесь.)

Партнеры могут получить максимальное количество баллов для заданной метрики, если они отвечают или превышают пороговые значения, указанные ниже. Частичные баллы могут выставляться для всех метрик, за исключением функционального роста консультанта, например, разработанного маркетингового партнера с $150 000 в процентах от нового дохода будет выставлять половину ($ 150000/$ 300000) максимум 10 PTS.

## <a name="customer-to-partner-association"></a>Ассоциация "клиент-участник"

Вы получаете баллы за чистую новую прибыль, net-клиенты, новые крупные развертывания и ежемесячное увеличение активного пользователя для данного клиента в пределах определения метрики, только если вы являетесь поставщиком облачных решений (CSP), партнером по заявке на запись (КПОР). или цифровой партнер записи (DPOR (DPOR) для платной подписки, принадлежащей клиенту клиента.

Партнеры становятся КПОР, если они установили связь с клиентом на уровне подписки с помощью стимула помощника по Dynamics 365 Online Solution Advisor (OSA).

Партнер станет DPOR, если он установил связь с клиентом на уровне подписки в рамках продукта.

Если для данной подписки существует и КПОР, и DPOR, КПОР получает все кредитные данные для метрик PCI, связанных с подпиской.
