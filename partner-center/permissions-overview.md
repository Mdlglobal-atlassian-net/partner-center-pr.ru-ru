# <a name="assign-users-roles-and-permissions"></a>Назначение пользователям ролей и разрешений


Вы настроили профиль партнера, включая юридическое имя и адрес, сведения о поддержке, налоговые исключения файлов, сведения о банке и основное контактное лицо для вашей компании. Следующий шаг: пользователи смогут настроить пароли и роли, чтобы они могли начать работу в центре партнеров с вами.

## <a name="set-up-your-employees-to-work-in-partner-center"></a>Настройка сотрудников для работы в центре партнеров

Вы определяете типы доступа пользователей к партнерским центрам по ролям и разрешениям, которые им предоставлены. Роли связаны с программами, в которых участвует ваш бизнес. Например, если ваш бизнес является бизнес-службой поставщика облачных решений (CSP), вы будете иметь не только стандартные роли управления клиентами Azure AD, такие как глобальный администратор, но и для программы CSP нужны специальные роли. Каждая программа имеет определенные роли.

>[!Note]
> Роли клиента Azure Active Directory (AAD) включают в себя роли глобального администратора, администратора пользователей и CSP. Роли, отличные от AAD, — это роли, которые не управляют клиентом, и включают администратора MPN, администратора бизнес-профиля, администратора ссылок, поощрения администратора и стимулического пользователя.
