# <a name="assign-users-roles-and-permissions"></a>Назначение пользователям ролей и разрешений


Вы настроили профиль партнера, включая юридическое название и адрес, сведения о поддержке, налог исключения файла, bank info и основного контактного лица для вашей компании. Следующий шаг: настройка с паролями, а также роли, чтобы они могли начать работу в центре партнеров с вами пользователи.

## <a name="set-up-your-employees-to-work-in-partner-center"></a>Настройка сотрудников для работы в центре партнеров

Вы определите типы доступа пользователей к центра партнеров, роли и разрешения, которые вы не предоставите им. Роли, связаны с исполняемом формате, который вашего бизнеса участвует в. Например, если вашего бизнеса — это поставщик облачных решений (CSP), вы не только будет standard Azure AD роли управления, такие как глобальный администратор клиента, но потребуется роли относятся к программе CSP. Каждая программа имеет роли, относящиеся к ней.

>[!Note]
> Роли клиента Azure Active Directory (AAD) включают глобального администратора, администратора пользователей и ролей CSP. Роли AAD не включать администратора MPN, бизнеса профиль администратора, администратора ссылок, стимулирования администратора и стимулирования пользователя.
