# <a name="moving-from-pmc-to-partner-center"></a>Переход с PMC на центр партнеров

Мы упростили бизнес с нами, представляя единый веб-сайт (центр партнеров), который выступает в качестве центральной точки взаимодействия. Все, что вы сделали в центре участия в партнерских организациях (PMC), можно выполнить на панели мониторинга в центре партнеров. 

Кроме того, можно выполнять множество других задач, не закрывая один веб-сайт. Однако некоторые термины и функции могут выглядеть иначе. Чтобы устранить любые первоначальные проблемы, связанные с тем, где и что это такое, воспользуйтесь обзором веб-узла панели мониторинга.

В этой таблице представлены некоторые различия между PMC и центром партнеров.

## <a name="renewing-your-microsoft-partner-network--membership"></a>Продление членства в Microsoft Partner Network

|**PMC**   |**Центр партнеров**|
|----------------------|:-----------------------------|
|Продление начато 90 до годовщины и должно быть завершено до даты юбилея| Партнеры могут продлить время начала дня после юбилея и до 30 дней после юбилея.|

## <a name="account-structure"></a>Структура учетной записи

|**PMC**   |**Центр партнеров**|
|----------------------|:-----------------------------|
|Штаб-квартира и местоположения — каждый из них вычисляется отдельно. Оценка компетенций была выполнена на локальном уровне|Одна глобальная компания (например, глобальная учетная запись партнера), включая расположения, вычисленные как единое целое; данные о производительности и навыках, агрегированные на уровне PGA; включает несколько представлений профилей для таких программ, как профиль партнера и бизнес-профиль для ссылок и маркетинга.|

## <a name="sign-in"></a>Войти

|**PMC**   |**Центр партнеров**|
|----------------------|:-----------------------------|
|Может использовать учетные данные учетная запись Майкрософт (MSA) или личной учетной записиjoe@outlook.com|Необходимо использовать учетные данные рабочей учетной записиjoe@joescompany.com(.)|

## <a name="user-roles"></a>Роли пользователя

|**PMC**   |**Центр партнеров**|
|----------------------|:-----------------------------|
|У вас много ролей в PMC, которые не указаны в центре партнеров|Администратору, который выполняет перемещение в центр партнеров, автоматически назначается роль администратора MPN, администратор учетной записи и ссылки. Затем они могут назначать роли пользователей другим пользователям.|
|Пользователи управляются на уровне расположения|Управление пользователями осуществляется на уровне компании (PGA), а не на уровне расположения. Исключением является администратор поощрения, который работает на уровне расположения.|
|   |Центр партнеров имеет два обширных набора ролей: роли, которые администрируют клиент Azure AD, и роли, которые управляют бизнесом компании. Вы можете упорядочить роли таким образом, чтобы они были осмысленными для вашей компании. После того как пользователь сможет выполнять все или многие люди, ему могут быть назначены отдельные роли и разрешения. 

## <a name="how-competencies-and-benefits-are-accounted-for"></a>Как учитывается компетенции и преимущества

|**PMC**   |**Центр партнеров**|
|----------------------|:-----------------------------|
|Начисляется на расположение и управляется на месте|Преимущества применяются ко всей Организации, включая администрирование преимуществ. Тем не менее, вы можете управлять преимуществами так, как это лучше подходит для вашей компании. |
|Может иметь дополнительные наборы средств для получения дополнительных преимуществ (Абткс), пока они не будут сняты в октябре 2018.|Нет Абткс; один MAPs для каждой компании; одна серебристая компетенция для каждой компании; Одна из компетенций Gold для каждой компании|
|Доступ к преимуществам при цифровом скачивании партнеров (PDD) |Все преимущества доступны в центре партнеров|
|Компетенции и преимущества, распределенные по нескольким расположениям|Компетенции и преимущества всех ваших местоположений консолидируются на уровне компании (PGA) и будут храниться до даты юбилея. На этом этапе необходимо приобрести или продлить уровень компании. Производительность и навыки, а также компетенции объединяются глобально|
|Утверждения ваучера по программе Software Assurance выполняются в средстве Валикатион и Редмптион (ВВР).|Теперь вы можете получать доступ и управлять ваучерами обучения Software Assurance (САТВ) и (или) развернутыми службами планирования (DPS) в центре партнеров.  Устаревшее средство ВВР будет списано 1 октября 2019 г.  |

## <a name="visual-studio-benefits-and-msa"></a>Преимущества и MSA в Visual Studio

|**PMC**   |**Центр партнеров**   |
|-----------------|:-----------------|
|Выделение преимуществ Visual Studio для MSA|Преимущества Visual Studio, выделенные для MSAs, будут учитываться и сохранены.|
||Выделения MSA Visual Studio будут сохранены после продления в центре партнеров.|
||В центре партнеров партнер может добавлять рабочие учетные записи и учетные записи гостевых пользователей, которые используют MSA из того же клиента, на котором партнер MPN Администратор в клиенте Azure AD. Если партнер является глобальным администратором в нескольких клиентах Azure AD, а все эти клиенты связаны с одной и той же учетной записью центра партнеров, то участнику разрешается добавлять пользователей для всех этих клиентов в преимущества Visual Studio и распределения на основе использования Azure. Хотя гостевым пользователям можно назначать подписки на основе использования Visual Studio администратором MPN или глобальным администратором, гостевые пользователи не могут войти в центр партнеров с помощью MSA. Однако гостевые пользователи могут входить в Azure и Visual Studio для проверки и использования их назначенных преимуществ. |
