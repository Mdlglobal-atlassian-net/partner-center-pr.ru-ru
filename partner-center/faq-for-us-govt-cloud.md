# <a name="frequently-asked-questions-for-partner-center-for-microsoft-cloud-for-us-government"></a>Ответы на вопросы о Центре партнеров для Microsoft Cloud for US Government

**Применяется к**

-  Центр партнеров для Microsoft Cloud for US Government

## <a name="frequently-asked-questions"></a>Вопросы и ответы 

В этом разделе приведены ответы на часто задаваемые вопросы о программе поставщиков облачных решений (CSP) для Microsoft Cloud для правительства США Для получения дополнительных сведений вы также можете обратиться к своему менеджеру по работе с клиентами. 

### <a name="common"></a>Общие

**Что такое Microsoft Cloud для государственных организаций США?**

Платформа Microsoft Cloud для правительства США, изначально разработанная специально под нужды правительственных организаций США, предлагает большой выбор служб облачных вычислений для клиентов из государственного сектора — от крупных федеральных агентств до администраций небольших городов. Чтобы лучше понять особые потребности своих клиентов, корпорация Майкрософт осуществила крупные инвестиции в центры обработки данных, а также провела работу по приведению своих услуг в соответствие с федеральными законами США, законами штатов, предписаниями и требованиями. 

**Кто имеет право по программе CSP для Microsoft Cloud для государственных организаций США?**

Партнеры в США, работающие с федеральными, региональными и муниципальными государственными организациями, могут принять участие в программе CSP для Microsoft Cloud for US Government. Корпорация Майкрософт применяет строгие проверки для определения соответствия партнеров требованиям, которые хотят принять участие в программе Microsoft Cloud для правительства США. Чтобы получить право участия, вам необходимо подтвердить, что вы работаете с государственными учреждениями. Процесс регистрации в программе CSP для Microsoft Cloud для правительства США был модифицирован с целью сбора дополнительных данных для проверки. К типам подтверждений, в том числе, относятся номера государственных контрактов и спонсорские письмо от ваших клиентов из числа государственных учреждений.

**Клиент цены отличаются от коммерческих CSP?**

Нет. Стоимость участия включает в себя дополнительную плату, что подразумевает уникальность дополнительных возможностей программы Microsoft Cloud для правительства США.

**Совпадает модели распродаж партнера?**

Да. Модель скидок для партнеров в рамках программы CSP для Microsoft Cloud для правительства США аналогична модели скидок для партнеров в рамках коммерческой программы CSP Commercial.

**Коммерческие CSP-это существующая программа партнера. Чем отличается программе CSP для облака Microsoft для государственных организаций?**

Программа CSP для Microsoft Cloud для правительства США отличается от коммерческой программы CSP Commercial по следующим аспектам.

-   Доступность:  Программе CSP для облака Microsoft для государственных организаций доступна только для партнеров в Соединенных Штатах, которые служат федеральных, state, государственных и местных учреждений осуществляющих правительственные. Корпорация Майкрософт применяет проверки для определения соответствия партнеров требованиям, которые хотят принять участие в программе Microsoft Cloud для правительства США.

-   Подключение: Партнерам необходимо подключение к программе CSP для Microsoft Cloud для государственных организаций США отдельно от коммерческих CSP.

-   Доступность продуктов: Azure для государственных организаций и служб на основе рабочих мест, такие как Office 365, Enterprise Mobility + Security и Dynamics 365 доступны в программе CSP для Microsoft Cloud для государственных организаций. 

-   Сжатие: Соглашения торгового посредника Microsoft Cloud (MCRA) была обновлена в соответствии с условиями уникальный для государственных организаций.

-   Функциональные возможности центра партнеров: Некоторые возможности центра партнеров не будет включен для программы CSP для Microsoft Cloud для государственных организаций США в соответствии нормативным и законодательным требованиям. Партнеры могут использовать Azure Active Directory (Azure AD) или [API Graph](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS) для выполнения задач, связанных с управлением пользователями и группами. Функция запроса связи для поддержки сценариев со множеством каналов и партнеров будет реализована после запуска, когда службы на основе числа мест станут доступны в программе CSP для Microsoft Cloud для правительства США.

**Я не являюсь партнером CSP еще. Где можно найти дополнительные сведения о том, как стать партнером CSP?**

Дополнительные сведения о моделях регистрации, доступных партнерам, см. в разделе [Поставщик облачных решений в Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider).

**Как работает лицензирование?**

*Соглашения торгового посредника Microsoft Cloud (MCRA)*

Соглашение MCRA позволяет партнерам продавать решения с помощью CSP. Версия этого соглашения для Microsoft Cloud для правительства США принимается при регистрации в Microsoft Cloud для правительства США. Эта версия содержит дополнение для облака для государственных организаций США и приложение А (после стандартного соглашения торгового посредника для облачной платформы).

Соглашение MCRA для Microsoft Cloud для правительства США отличается от стандартного соглашения MCRA двумя аспектами.

-   Дополнение для облака для государственных организаций США описывает дополнительные обязанности партнеров по транзакциям с облачными службами для правительства США.

-   Приложение А содержит соглашения Microsoft Cloud (соглашение Microsoft Cloud), должны быть включены в каждый контракт для службы облака для государственных организаций США, между CSP и клиентом.

*Как приложение, А в MCRA соглашение Microsoft Cloud (соглашение Microsoft Cloud)*

Поставщикам CSP необходимо включать приложение А в государственные контракты, заключенные между CSP и клиентами, приобретающими Microsoft Cloud для правительства США. Соглашение публикуется в Центре партнеров и доступно на английском языке.

Соглашение Microsoft Cloud для облачных служб для правительства США отличается от стандартного соглашения Microsoft Cloud двумя аспектами.

-   Оно должно быть включено в контракт CSP с клиентом.

-   Оно было адаптировано в соответствии с федеральными, региональными и муниципальными законами и нормативными актами.

*Соглашение Microsoft Cloud распространителя (MCDA)*

Дистрибьюторское соглашение Microsoft Cloud (MCDA) позволяет дистрибьюторам продавать решения в рамках программы CSP. Версия MCDA для облачных служб для правительства США содержит дополнение для облака для государственных организаций США и приложение А (после стандартного соглашения торгового посредника для облачной платформы (MCRA)).

Соглашение MCDA для Microsoft Cloud для правительства США отличается от стандартного соглашения MCDA двумя аспектами.

-   Дополнение Microsoft Cloud для правительства США описывает дополнительные обязанности партнеров по транзакциям с облачными службами для правительства США.

-   Приложение А содержит соглашения Microsoft Cloud (соглашение Microsoft Cloud), должно быть отражено в каждый контракт для служб облака для государственных организаций США, между CSP и своих клиентов.

**Что такое квалификации для продажи Microsoft Cloud для государственных организаций США?**

Критерии допуска к осуществлению продаж в рамках программы CSP для Microsoft Cloud для правительства аналогичны критериям допуска к осуществлению продаж в рамках коммерческой программы CSP Commercial. Этим критериям соответствуют как прямые партнеры, так и косвенные торговые посредники. Если вы утверждены в качестве косвенного торгового посредника в рамках коммерческой программы CSP Commercial, необходимость в повторном прохождении процедуры утверждения в качестве косвенного торгового посредника в рамках программы CSP для Microsoft Cloud для правительства США.

**Я являюсь партнером существующий. Что нужно сделать для продажи Microsoft Cloud для государственных организаций США?**

Партнерам необходимо присоединиться к программе CSP для Microsoft Cloud для правительства США, чтобы выполнить требования нормативных актов и соответствия. (Регистрация в программе CSP для правительства США осуществляется отдельно от регистрации в коммерческой программе CSP Commercial.) После регистрации вы сможете создать клиентскую базу и проводить сделки в рамках программы CSP для Microsoft Cloud для правительства США. Если вы используете API-интерфейсы, см. раздел [Разработка для Центра партнеров для Microsoft National Cloud](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS).

**Есть ли разница в поощрения для партнеров?**

Нет. Поощрения для партнеров в рамках программы CSP для Microsoft Cloud для правительства США аналогичны поощрениям для партнеров в рамках коммерческой программы CSP Commercial. Чтобы узнать больше о поощрениях для партнеров, войдите на [портал Microsoft Partner Network](https://partner.microsoft.com/membership/partner-incentives).

**Существуют ли все изменения на выставление счетов?**

Нет. Процедура выставления счетов в рамках программы CSP для Microsoft Cloud для правительства США аналогична процедуре выставления счетов в рамках коммерческой программы CSP Commercial. Дополнительные сведения о выставлении счетов см. в ресурсах [Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider/resources).

**Для данных, размещенных в коммерческих центров обработки данных это можно изменить в Microsoft Cloud для государственных организаций США?**

В целях обеспечения целостности изоляции облака Microsoft Cloud для правительства США механизмы переноса имеющихся у клиента служб в облако Microsoft Cloud для правительства США не предусмотрены. Если клиент или партнер использует Azure, они будут отвечать за любое перемещение данных в новую среду в Microsoft Cloud для правительства США.


### <a name="microsoft-cloud-for-us-government-for-azure"></a>Microsoft Cloud for US Government для Azure

**Что такое Microsoft Cloud для государственных организаций США для Azure?**

Azure для государственных организаций — это облачная платформа, созданная на основе принципов безопасности, конфиденциальности, контроля, соответствия требованиям и прозрачности. Государственные организации получают физически изолированный экземпляр Microsoft Azure, использующий службы обеспечения безопасности и соответствия мирового уровня, необходимые для всех систем и приложений правительства США. Эти службы сертифицированы на соответствие стандартам FedRAMP и Министерства обороны США, соглашениям CJIS государственного уровня, позволяют выдавать соглашения HIPAA для деловых партнеров и поддерживают IRS 1075. Платформа Azure для государственных организаций, которой управляют проверенные граждане США, поддерживает множество гибридных сценариев для создания и развертывания решений в локальной или облачной среде. Государственные организации также могут воспользоваться преимуществами мгновенной масштабируемости и гарантированной доступности этой надежной облачной службы.

**Программа CSP для Microsoft Cloud для государственных организаций США предлагает всех служб Azure?**

В настоящий момент доступны следующие предложения: «инфраструктура как услуга» (IaaS), базовая платформа и решения для работы с данными. Схема обновляется регулярно, и мы предоставляем документацию, аналогичную документацию для коммерческих CSP в центре партнеров (требуется проверка подлинности). Вы можете просмотреть службы Azure для государственных организаций, доступные сейчас, но обратите внимание, что некоторые службы, такие как классические вычислительные, сетевые и облачные службы, а также службы хранения, не предназначены для CSP.

**Что нужно сделать, чтобы разработка с помощью программы CSP для Microsoft Cloud для государственных организаций США?**

Дополнительные сведения о разработке с помощью Центра партнеров для Microsoft Cloud для правительства США см. в разделе [Разработка с помощью Центра партнеров для облака для государственных учреждений](https://msdn.microsoft.com/library/partnercenter/mt789013.aspx#partner_center_msftcloudUS).

**Существуют ли все различия в API-интерфейсы?**

API-интерфейсы и требования к их интеграции в рамках программы CSP для Microsoft Cloud для правительства США аналогичны API-интерфейсам и требованиям к их интеграции в рамках коммерческой программы CSP Commercial. Дополнительные сведения о конечных точках в Microsoft Cloud для правительства США см. в разделе [URL-адреса REST Центра партнеров](https://msdn.microsoft.com/library/partnercenter/mt490977.aspx).

>[!NOTE]
>API-интерфейсы CREST будут недоступны в рамах программы CSP для Microsoft Cloud for US Government. Эти API-интерфейсы удалены из программы CSP и недоступны для Microsoft Cloud для правительства США. Для выполнения нормативных требований и требований соответствие следующие API-интерфейсы будут недоступны в рамках программы CSP для Microsoft Cloud для правительства США.
