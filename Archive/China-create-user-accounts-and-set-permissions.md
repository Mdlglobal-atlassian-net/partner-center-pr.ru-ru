# <a name="create-user-accounts-and-set-permissions"></a>Создание учетных записей пользователей и определение разрешений


**Применяется к**

-   Центр партнеров, предоставляемый 21Vianet


Создание учетных записей пользователей для сотрудников, которым необходимо выполнить задачи в центре партнеров. Только глобальные или Администраторы пользователей управления для добавления пользователей и назначение ролей. В **параметры учетной записи** &gt; **Управление пользователями**, можно добавить учетные записи пользователей и назначить или изменить разрешения.

**Добавить нового пользователя**

1.  В центре партнеров, перейдите в меню панели мониторинга &gt; **параметры учетной записи** &gt; **Управление пользователями**.

2.  Выберите **Добавить пользователя**.

3.  Введите полное имя пользователя и уникальный адрес электронной почты.

4.  Выберите уровень доступа и роли сотрудника. Доступ к центру партнеров и разрешения основаны на роли, который позволяет контролировать конкретные задачи, которые могут выполнять каждого сотрудника. Дополнительные сведения о каждой роли, которые можно сделать, см. в разделе [настроить разрешения](#setuserpermissions).

5.  Выберите **Добавить** для создания учетной записи пользователя. Откроется страница подтверждения отображение разрешения и учетные данные, включая временный пароль нового пользователя.

    >**Внимание!**<br>Когда новые пользователи входят центре партнеров в первый раз, они вход временный пароль. Обязательно запомните или запишите временный пароль нового пользователя, отправить его позже. Нельзя будет получить доступ к еще раз. 

### <a href="" id="setuserpermissions"></a>Установка разрешений пользователя

Доступ к центру партнеров основан на ролях, это означает, что вы можете ограничить представление пользователя для отображения только те функции и функций, которые пользователь должен выполнить определенные задачи. Для каждого пользователя необходимо выбрать два параметра:

-   **Агента** параметр определяет, какого рода данные клиентов и компании, которые может видеть пользователь.

-   **Администратора** параметр определяет, какой объем прав доступа пользователя к функциям центра партнеров, функции и данные. 

    >**Примечание**<br>Значение по умолчанию должно быть **нет администратора** в большинстве случаев, если сотрудник не требует разрешений для выполнения определенных административных задач.
